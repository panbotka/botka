When a new version of the app is deployed, the PWA should detect it and prompt the user to refresh. Currently the service worker caches everything and the user has no way to trigger an update from within the app (especially on iOS where hard refresh is difficult).

## Current behavior
- PWA uses vite-plugin-pwa with `generateSW` mode
- Once cached, the app serves stale assets indefinitely
- On iOS there's no easy way to force a refresh — the user has to delete website data from Settings or remove and re-add the PWA

## Expected behavior
- When a new service worker is detected (new build deployed), show a banner/toast: "New version available" with a "Refresh" button
- Clicking refresh activates the new service worker and reloads the page
- The banner should be noticeable but not blocking

## Requirements

### Service worker update detection
- vite-plugin-pwa supports `registerType: 'prompt'` which gives control over when to activate the new SW
- When a new SW is waiting, fire a callback that the app can use to show the banner
- The `useRegisterSW` hook from `virtual:pwa-register/react` provides `needRefresh` state and `updateServiceWorker()` function

### Frontend — Update banner
- Show a fixed banner (top or bottom of screen) when `needRefresh` is true
- Content: "New version available" + "Refresh" button
- Clicking "Refresh" calls `updateServiceWorker(true)` which activates the new SW and reloads
- Optional "Dismiss" button (but it will show again on next visit)
- Style: subtle but visible, matching app design (e.g. blue/indigo accent, zinc background)

### PWA config
- In `vite.config.ts`, ensure `registerType: 'prompt'` is set (not `'autoUpdate'`)
- This gives the app control over when to apply the update instead of silently updating in the background

### Registration
- In `main.tsx` or the app root, use `useRegisterSW` from `virtual:pwa-register/react`:
  ```tsx
  const { needRefresh: [needRefresh], updateServiceWorker } = useRegisterSW()
  ```
- Pass `needRefresh` and `updateServiceWorker` to the banner component

## Implementation Notes
- vite-plugin-pwa docs: https://vite-pwa-org.netlify.app/guide/prompt-for-update.html
- The key is switching from `autoUpdate` (or default) to `prompt` registerType
- Check current `vite.config.ts` for the PWA plugin config
- Existing tests must pass (`make check`)

## Safety
**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.**
