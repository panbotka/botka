The `claude-usage` command now returns `age_seconds` (int) and `stale` (bool, true if data >15 min old) at the top level of its JSON output. Propagate these fields through the backend to the frontend and show a visual warning when usage data is stale.

## Current state

**claude-usage JSON** (top-level fields):
```json
{
  "fetched_at": "...",
  "age_seconds": 101,
  "stale": false,
  "data": { "five_hour": {...}, "seven_day": {...} }
}
```

**Backend** (`internal/runner/usage.go`):
- `claudeUsageResponse` struct only parses `data` — ignores `age_seconds` and `stale`
- `UsageInfo` struct has `LastChecked` (set to `time.Now()` on parse, not from API) but no `stale` or `age_seconds`

**Frontend** (`frontend/src/types/index.ts`):
- `UsageInfo` has `five_hour_pct`, `seven_day_pct`, `resets_at`, `last_checked` — no staleness info

**Frontend** (`frontend/src/pages/DashboardPage.tsx`):
- `UsageMeters` component shows two progress bars with percentages, no staleness indicator

## Requirements

### Backend — Parse new fields
- Add `AgeSeconds int` and `Stale bool` to `claudeUsageResponse` (top-level, not inside `data`)
- Add `AgeSeconds int` and `Stale bool` to `UsageInfo` struct
- Populate them from the parsed JSON in `parseUsageJSON`
- These fields get exposed via the existing runner status endpoint (`GET /api/v1/runner/status`) and refresh endpoint (`POST /api/v1/runner/usage/refresh`)

### Frontend — Update types
- Add `age_seconds: number` and `stale: boolean` to `UsageInfo` interface

### Frontend — Show stale warning in UsageMeters
- When `usage.stale` is `true`, show a visible warning in the API Usage box:
  - Change the box border/background to indicate stale data (e.g. amber/yellow tint)
  - Show a warning line like "Data is X min old" (computed from `age_seconds`)
  - Make the refresh button more prominent (e.g. pulse animation or highlight)
- When `usage.stale` is `false`, show nothing extra (current behavior)
- Optionally show `age_seconds` as subtle "Updated X min ago" text even when not stale

### Styling
- Follow existing design language (Tailwind, zinc palette, amber for warnings)
- Warning should be noticeable but not alarming
- Keep the box compact

## Implementation Notes
- `parseUsageJSON` in `usage.go` is where the JSON gets parsed — add the two fields to `claudeUsageResponse` at the top level (not inside `.Data`)
- `UsageInfo` is returned by `CurrentUsage()` and serialized to JSON for the API — adding fields there propagates automatically
- The `UsageMeters` component in `DashboardPage.tsx` (~line 109) is the only place to change in frontend
- Existing tests must pass (`make check`)
