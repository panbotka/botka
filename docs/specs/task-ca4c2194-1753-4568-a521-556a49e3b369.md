## Task 8: Claude Subprocess Runner and Context Assembly

Port Chatovadlo's Claude Code subprocess management and context assembly system to Botka.

### Context
This is the shared Claude Code subprocess layer used by the chat system. It spawns interactive Claude Code sessions with session continuity (--resume) and assembles multi-layer system prompts from workspace files (SOUL.md, USER.md, etc.), app memories, and project context.

This is SEPARATE from the task runner's executor (Task 5) which has its own spawn logic for batch execution. Both coexist ŌĆö chat uses `internal/claude/`, tasks use `internal/runner/executor.go`.

### Reference Files
- `/home/pi/projects/chatovadlo/internal/claude/runner.go` ŌĆö Run(), StreamEvent, parseEvent, EventKind constants
- `/home/pi/projects/chatovadlo/internal/claude/context.go` ŌĆö AssembleContext with 8-layer system prompt
- `/home/pi/projects/chatovadlo/internal/claude/registry.go` ŌĆö ProcessRegistry for tracking active chat processes

### What to Create

1. **`internal/claude/runner.go`** ŌĆö Port from Chatovadlo:
   - `StreamEvent` struct with Kind, Text, Thinking, ToolName, ToolInput, ToolID, ToolUseID, ToolContent, ToolIsError, ResultText, CostUSD, DurationMs, NumTurns, InputTokens, OutputTokens, IsError, ErrorMsg, Raw
   - `EventKind` enum: KindInit, KindContentDelta, KindContentDone, KindThinkingDelta, KindThinkingDone, KindToolUse, KindToolResult, KindResult, KindError, KindIgnored
   - `RunConfig` struct: ClaudePath, WorkDir, SessionID, Resume, Model, SystemPromptFile, Name
   - `Run(ctx, cfg, prompt) <-chan StreamEvent` ŌĆö spawns claude subprocess with stream-json output, parses NDJSON events
   - `parseEvent`, `parseStreamEvent`, `parseAssistantEvent`, `parseResultEvent` ŌĆö event parsers
   - `Compact(ctx, cfg) <-chan StreamEvent` ŌĆö send /compact command
   - `TitleFromContent(content) string` ŌĆö truncate content for auto-titles

   Update import path from `chatovadlo` to `botka`. No other changes needed ŌĆö this code is self-contained.

2. **`internal/claude/context.go`** ŌĆö Port from Chatovadlo:
   - `ContextConfig` struct: OpenClawWorkspace, ContextDir
   - `AssembleContext(ctx, cfg, threadID, memoryRepo, systemPrompt, folderClaudeMD, messages) (string, error)`
   - 8 layers: SOUL.md, USER.md, MEMORY.md, recent daily notes, app memories, thread system prompt, project CLAUDE.md, conversation history
   - `readFileIfExists`, `recentDailyNotes` helpers

   NOTE: The memoryRepo parameter currently depends on Chatovadlo's repository package. Since Botka uses GORM instead of raw pgx, you need to change the memory query parameter. Options:
   - Accept a `func(ctx context.Context) (string, error)` callback instead of a repo
   - Or accept a `*gorm.DB` and query memories directly
   - Choose whichever is cleaner. The query is simple: SELECT content FROM memories, join with newlines.

3. **`internal/claude/registry.go`** ŌĆö Port from Chatovadlo:

   - `Register(threadID, title, cancelFunc)`, `Unregister(threadID)`, `List() []ProcessInfo`, `Kill(threadID)`
   - Global `Registry` variable
