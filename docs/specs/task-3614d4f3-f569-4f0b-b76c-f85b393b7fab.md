# Diff viewer for file edit tool calls in chat

Show a readable diff view when Claude uses the `Edit` or `Write` tools, instead of raw JSON input.

## Requirements

### Detection
- Identify tool calls where `name` is `Edit` or `Write` in the streaming UI
- Parse the `input` JSON to extract relevant fields

### Edit tool diff
- For `Edit` tool calls, input contains `file_path`, `old_string`, `new_string`
- Render a side-by-side or unified diff view showing old_string → new_string
- Show the file path as a header above the diff
- Syntax highlight if possible (or at minimum use monospace font with red/green lines)

### Write tool display
- For `Write` tool calls, input contains `file_path` and `content`
- Show as a new file creation with all lines in green (addition)
- Show the file path as header

### UI
- Render inline within the tool call display area (where tool calls currently show)
- Collapsible — show a summary line (e.g. "Edit: src/foo.go (+3 -2)") with expand/collapse toggle
- Default to collapsed for large diffs (>20 lines), expanded for small ones
- Use standard diff colors: green background for additions, red for deletions
- Monospace font for diff content
- Follow existing app design language (zinc palette, Tailwind)

### Fallback
- For all other tool calls, keep the current display (raw name + input)
- If input parsing fails, fall back to raw display

## Implementation Notes

- No external diff library needed — `Edit` already provides old/new strings, just render them as unified diff
- The tool call data is available in SSEContext as `{ id, name, input }` where input is `Record<string, unknown>`
- Existing tests must pass (`make check`)

## Safety

**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.**
