Add a project selector to the chat UI so users can see which project a thread is connected to, assign a project, and change it.

## Requirements

1. **Visible project indicator** — On the chat page, show the currently assigned project (or "No project" if none). This should be visible without opening a menu.

2. **Project picker** — Clicking the indicator opens a dropdown/popover listing all available projects (fetched from `GET /api/v1/projects`). The list should include a "None" option to unassign.

3. **Assign/change project** — Selecting a project calls `updateThreadProject(threadId, projectId)` from `frontend/src/api/client.ts:239` which hits `PUT /api/v1/threads/:id/project`.

4. **Unassign project** — Selecting "None" calls `updateThreadProject(threadId, null)`.

5. **Update local state** — After changing the project, update the thread in local state so the sidebar badge and chat header reflect the change immediately without a page reload.

## Existing code to use

- **API client function already exists:** `updateThreadProject` in `frontend/src/api/client.ts:239` — use it, don't create a new one.
- **Projects list:** There's already a projects fetch in the app (used by TaskForm and ThreadSidebar). Reuse the same data source / hook.
- **Thread type** already has `project_id?: string` in `frontend/src/types/index.ts:87`.
- **Sidebar** already shows a project badge on threads (`ThreadSidebar.tsx:299-302`) — keep that working.

## Placement

Place the project selector in the chat header area (where thread title / model info is shown). Use a compact design — e.g. a small chip/badge that shows the project name, clickable to open the picker. Use Lucide icons (`FolderOpen`, `FolderClosed`, or `GitBranch`) for visual consistency with the rest of the app.

## Style

- Follow existing Tailwind CSS patterns in the codebase
- Match the look and feel of other selectors in the app (e.g. model selector, persona selector if any)
- Keep it minimal and unobtrusive

- The backend endpoint `PUT /api/v1/threads/:id/project` already works — no backend changes needed.
