## Goal

When a user renames a thread in the UI, the Claude Code session name should update to match. Currently, `--name` is only passed when creating a new session (`!cfg.Resume`), so resuming a session keeps the old name (e.g., "New Chat") even after the thread is renamed.

## Problem

In both `internal/claude/runner.go` (line ~120) and `internal/claude/pool.go` (line ~512), the `--name` flag is conditionally skipped on resume:

```go
if cfg.Name != "" && !cfg.Resume {
    args = append(args, "--name", cfg.Name)
}
```

This means a session created as "New Chat" keeps that name forever, even after the user renames the thread.

## Fix

Remove the `!cfg.Resume` condition in both places so `--name` is always passed:

```go
if cfg.Name != "" {
    args = append(args, "--name", cfg.Name)
}
```

Both files need the same change:
1. `internal/claude/runner.go` — `RunStream()` function, the args building section
2. `internal/claude/pool.go` — `buildStreamArgs()` function
