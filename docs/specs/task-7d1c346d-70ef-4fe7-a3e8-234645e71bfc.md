## Task 9: Thread and Chat Message Handlers

Port Chatovadlo's thread management and chat streaming handlers to Botka, converting from Chi+pgx to Gin+GORM.

### Context
This is the core chat functionality. Threads hold conversations with Claude Code. Messages support branching (parent_id tree). Chat responses stream via SSE from spawned Claude Code processes. File uploads are supported.

### Reference Files
- `/home/pi/projects/chatovadlo/internal/handler/chat.go` — SendMessage, Regenerate, EditMessage, Branch, SwitchBranch, streamResponse, ClearSession, NewSession, saveUploadedFile
- `/home/pi/projects/chatovadlo/internal/handler/thread.go` — Thread CRUD, pin, archive, model, usage, folder assignment
- `/home/pi/projects/chatovadlo/internal/handler/file.go` — File serve/download
- `/home/pi/projects/chatovadlo/internal/repository/thread.go` — Thread SQL queries (for GORM conversion)
- `/home/pi/projects/chatovadlo/internal/repository/message.go` — Message SQL queries including GetActivePath, GetLastMessageInPath, SetBranchSelection, DeleteBranch

### What to Create

1. **`internal/handlers/thread.go`** — Thread CRUD using GORM:
   - `GET /api/v1/threads` — List threads (with `?archived=true` filter). Include last message preview, unread indicator, tags. Order: pinned first, then by updated_at DESC.
   - `POST /api/v1/threads` — Create thread (optional: persona_id, model, project_id)
   - `GET /api/v1/threads/:id` — Get thread with full message tree (active path + fork points). This is the most complex query — must walk the message tree using branch_selections to determine the active path.
   - `PUT /api/v1/threads/:id` — Rename thread
   - `DELETE /api/v1/threads/:id` — Delete thread and all messages (CASCADE)
   - `PUT /api/v1/threads/:id/pin` / `DELETE /api/v1/threads/:id/pin`
   - `PUT /api/v1/threads/:id/archive` / `DELETE /api/v1/threads/:id/archive`
   - `PUT /api/v1/threads/:id/model` — Set AI model for thread
   - `PUT /api/v1/threads/:id/project` — Assign thread to project (replaces folder assignment)
   - `GET /api/v1/threads/:id/usage` — Token usage stats
   - `DELETE /api/v1/threads/:id/messages` — Clear all messages
   - `PUT /api/v1/threads/:id/tags` — Set thread tags (array of tag IDs)

   IMPORTANT: Convert all raw pgx SQL to GORM. For complex queries (active path, message tree), use `db.Raw()` with SQL.

2. **`internal/handlers/chat.go`** — Chat streaming using Gin:
   - `POST /api/v1/threads/:id/messages` — SendMessage. Supports both JSON and multipart/form-data (for file uploads). Saves user message, then streams Claude response via SSE.
   - `POST /api/v1/threads/:id/regenerate` — Delete last assistant message, re-stream
   - `POST /api/v1/threads/:id/messages/:messageId/edit` — Create branch at edited message, stream new response
   - `POST /api/v1/threads/:id/branch` — Create new branch from any message
   - `PUT /api/v1/threads/:id/active-branch` — Switch active branch selection
   - `POST /api/v1/threads/:id/session/clear` — Clear Claude session ID
   - `POST /api/v1/threads/:id/session/new` — Generate new session ID

   The `streamResponse` function is critical — it:
   1. Determines session ID (new or resume)
   2. Gets working directory from project (was folder)
   3. Assembles context for new sessions (calls claude.AssembleContext)
   4. Spawns Claude via claude.Run()
   5. Registers process in claude.Registry
   6. Streams events as SSE: content deltas, thinking, tool_use, tool_result, usage, errors
   7. Saves assistant response to DB on completion
   8. Auto-generates title from first message

   For Gin SSE: use `c.Writer` as `http.ResponseWriter`, get `http.Flusher` from it, write `data: ...\n\n` format manually (same as Chatovadlo does with Chi).

3. **`internal/handlers/file.go`** — File upload/serve:
   - `GET /api/v1/files/:id` — Serve uploaded file
   - `GET /api/v1/files/:id/download` — Download with Content-Disposition header
   - File saving logic in chat handler (saveUploadedFile)

4. **Helper functions needed:**
   - `paramInt64(c *gin.Context, name string) (int64, error)` — parse route param as int64
   - `paramUUID(c *gin.Context, name string) (uuid.UUID, error)` — parse route param as UUID
   - Message tree walking functions using GORM (GetActivePath, GetLastMessageInPath, etc.)

5. **Wire all routes** in main.go setupRouter.

### Key GORM Conversion Notes
- Thread list with last message: use GORM Preload or a subquery
- Active path (message tree walk): use recursive CTE in `db.Raw()` — this is complex SQL that's better written as raw SQL
- Branch selection: simple GORM `FirstOrCreate` + `Save`
- Attachments: GORM `Create` + `Find` with conditions
