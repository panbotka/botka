# Fuzzy Search in Project Picker

Add a text input with fuzzy search to the project picker dropdown so users can quickly find projects by typing.

## Requirements

- Add a text input field at the top of the project picker dropdown that auto-focuses when the dropdown opens
- Filter the project list as the user types, using fuzzy matching against project `name` (e.g. typing "bot" matches "botka", typing "sd" matches "sdhveselice")
- Fuzzy matching should be case-insensitive and match non-contiguous characters (e.g. "mfx" matches "memofix")
- Sort matched results by match quality — exact prefix matches first, then substring matches, then fuzzy matches
- The "No project" option should always remain visible at the top regardless of search input
- Show a "No matches" message when the search yields no results
- Clear the search input when the dropdown closes
- Preserve all existing behavior: click-outside-to-close, current project highlighting, project selection callback
- Keep the dropdown compact — the input should be minimal (no label, just a placeholder like "Search...")
- Implement fuzzy matching inline in the component (no external library needed) — a simple scoring function that checks if all characters of the query appear in order in the target string

## Implementation Notes

- Component to modify: `frontend/src/components/ProjectPicker.tsx`
- Follow existing patterns in the codebase (Tailwind classes, zinc/amber color scheme)
- The fuzzy match function can be a small helper at the top of the file
