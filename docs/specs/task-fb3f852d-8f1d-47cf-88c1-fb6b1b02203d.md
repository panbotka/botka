# Bulk Purge Task Execution Outputs

Add a maintenance action in the Settings page that lets the user purge all stored raw output from completed task executions. The `task_executions.raw_output` column stores full Claude Code session output as TEXT and can grow large over time.

## Requirements

### Backend

- Add a new endpoint: `DELETE /api/v1/settings/task-outputs`
- The endpoint sets `raw_output = NULL` on ALL rows in `task_executions` where `raw_output IS NOT NULL`
- Return the number of purged rows in the response: `{"data": {"purged": 42}}`
- Register the route alongside existing settings routes in the router setup

### Frontend

- Add a new section in the Settings page called "Maintenance" (or "Data Management")
- Place it after the existing settings sections (below Server settings / Passkeys / etc.)
- Show a button labeled "Purge task outputs" with a Trash2 icon
- Show informational text explaining what this does: something like "Delete stored raw output from all task executions. This frees database space but you won't be able to view past task outputs."
- Before executing, show a confirmation dialog (window.confirm is fine) — "This will permanently delete all stored task outputs. Continue?"
- After successful purge, show a toast or inline success message with the count: "Purged output from X executions"
- The button should show a loading spinner while the request is in progress
- Add the API call to `frontend/src/api/client.ts`

### API Client

- Add `purgeTaskOutputs()` function to `frontend/src/api/client.ts` that calls `DELETE /api/v1/settings/task-outputs`

## Edge Cases

- If there are no outputs to purge (0 rows affected), still return success with `purged: 0` — the UI should show "No task outputs to purge" or similar
- The endpoint does NOT delete task executions themselves — only clears the `raw_output` column
