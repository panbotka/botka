# Frontend UI for Remote Directory Selection

Add UI support for selecting remote working directories on Box when configuring a thread, so users can choose between local (RPi) and remote (Box) execution.

## Requirements

### Thread directory selector

Modify the existing thread project/directory selector to support remote directories:

- Currently, threads can be assigned a project (which sets the working directory)
- Extend this to allow selecting remote directories on Box
- Group directories into two sections in the selector:
  - **Local (RPi):** existing projects from `/home/pi/projects/`
  - **Remote (Box):** directories from Box (fetched or configured)

### Remote directory discovery

Add an API endpoint to list available directories on Box:

- `GET /api/v1/box/projects` — returns list of project directories on Box
- Backend implementation: runs `ssh box@box ls /home/box/projects/` (or reads from a configured path)
- Returns: `{"data": [{"name": "myapp", "path": "box:/home/box/projects/myapp"}]}`
- If Box is offline, return empty list with a note that Box is unavailable
- Cache the result for a few minutes to avoid repeated SSH calls

### Visual indicators

- Threads running on Box should show a small "Box" badge or computer icon in the thread list
- The thread header/settings should indicate "Running on Box" when a remote directory is selected
- When Box is offline and a remote thread is selected, show a warning that Box needs to be started

### Box status indicator

- Show Box connection status somewhere accessible (sidebar or settings)
- States: Online (green), Offline (gray), Booting (yellow/pulsing)
- Optional: "Wake Box" button that triggers `boxon` via API

### API integration

Add API client methods:

- `getBoxProjects()` → `GET /api/v1/box/projects`
- `getBoxStatus()` → `GET /api/v1/box/status` (returns online/offline/booting)
- `wakeBox()` → `POST /api/v1/box/wake`

### Design

Follow existing design patterns. Use Lucide icons (Server or Monitor for Box indicator). Keep it consistent with how local projects are displayed — just add the remote section.
