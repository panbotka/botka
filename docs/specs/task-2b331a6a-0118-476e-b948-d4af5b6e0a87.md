The bottom navigation bar gets clipped by the rounded screen corners on iPhone 17 Pro **in PWA standalone mode only**. In Safari, the browser toolbar at the bottom provides enough offset so it's fine. In PWA mode (installed to home screen), there's no browser toolbar and the "Dashboard" label is cut off — only "Das" is visible in the corner.

## Root cause

The viewport meta tag is missing `viewport-fit=cover`:
```html
<!-- Current (index.html line 5): -->
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no" />
```

Without `viewport-fit=cover`, the browser restricts the layout to the "safe" rectangle and `env(safe-area-inset-bottom)` returns `0` — so the `paddingBottom` in `BottomNav.tsx` has no effect. In PWA standalone mode there's no browser chrome to compensate.

## Requirements

### Fix viewport meta tag
- Add `viewport-fit=cover` to the viewport meta tag in `frontend/index.html`:
  ```html
  <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no, viewport-fit=cover" />
  ```

### Verify safe-area padding
- `BottomNav.tsx` already has `paddingBottom: 'env(safe-area-inset-bottom, 0px)'` — this should start working once `viewport-fit=cover` is set
- If it's still not enough padding after the fix, add a small extra offset (e.g. `calc(env(safe-area-inset-bottom, 0px) + 4px)`) to ensure labels are comfortably above the rounded corners
- Test that all 6 tab labels ("Dashboard", "Chat", "Tasks", "Cost", "Settings", "Help") are fully visible in PWA mode

### Check other safe-area consumers
- `ChatPage.tsx` line 259 also uses `env(safe-area-inset-bottom)` — verify it also benefits from the fix
- Check that `viewport-fit=cover` doesn't cause issues with top content (status bar area) — the `apple-mobile-web-app-status-bar-style` is already `black-translucent` so this should be fine, but verify content doesn't render behind the notch/Dynamic Island

### Scope
- Only affects PWA standalone mode — Safari with its bottom toolbar is fine
- Test as installed PWA on iPhone (add to home screen)

## Implementation Notes
- This is a one-line HTML fix + verification
- `viewport-fit=cover` tells the browser to extend the page into the notch/rounded corner areas, and `env(safe-area-inset-*)` values become non-zero to indicate the unsafe regions
- The PWA manifest `"display": "standalone"` removes all browser chrome, so safe area insets are the only protection
- Existing tests must pass (`make check`)
