# Style URL Links in User Messages for All Themes

The previous task added clickable URL links in user messages, but the styling needs improvement. The current blue link color clashes with dark themes and looks out of place on user message bubbles.

## Requirements

In `frontend/src/components/MessageBubble.tsx` (or wherever the linkified user message URLs are rendered), update the link styling:

- **Text color should inherit from the parent** — do NOT use blue or any other explicit color. User message bubbles have white text on a colored background; links should stay white too.
- **Subtle underline:** `underline decoration-current/40 underline-offset-2` — a soft underline using the current text color at reduced opacity
- **Hover:** `hover:decoration-current/70` — slightly stronger underline on hover
- **Cursor:** `cursor-pointer` (should already be default for `<a>` tags)
- **Overflow:** Add `overflow-wrap: anywhere` or Tailwind `break-all` on links so very long URLs don't break the layout

This must look good in ALL 4 themes: light, dark, dark-green, dark-blue. Since the link inherits parent color and only uses a subtle underline, it will work universally.

## Files to Change

- The file where user message link elements are rendered — likely `frontend/src/components/MessageBubble.tsx` or a utility in `frontend/src/utils/` (check what the previous task created)

## Verification

- Open a chat thread with URLs in user messages
- Switch between all 4 themes (light, dark, dark-green, dark-blue)
- Links should be subtly underlined but not a different color from surrounding text
- Hover should make the underline slightly more visible
- Very long URLs should wrap without breaking the layout
