## Problem

The Dashboard page (`/`) currently shows too much clutter. The user wants a clean stats-focused dashboard showing only interesting task statistics at a glance.

## Requirements

Replace the current Dashboard content with a clean statistics overview:

### Key Stats to Show
- **Running** tasks count (currently active)
- **Queued** tasks count (waiting to run)
- **Pending** tasks count (not yet queued)
- **Done** tasks count (completed)
- **Failed** tasks count
- **Needs Review** tasks count
- **Total** tasks count

### Additional Interesting Stats (if data available)
- Tasks completed today / this week
- Success rate (done vs failed percentage)
- Most active project (by task count)
- Runner status (running/stopped, current workers)
- API usage stats if available

### Design
- Use stat cards with large numbers and labels
- Color-code by status (green for done, red for failed, blue for queued, yellow for running, etc.)
- Keep it minimal and scannable — no tables, no task lists, just numbers
- Responsive grid layout (2-3 columns on desktop, 1 on mobile)
- Proper dark mode support

## Files

- `frontend/src/pages/DashboardPage.tsx` — rewrite with stats-only layout
- `frontend/src/api/client.ts` — add API calls if new endpoints are needed
- Backend: add a `/api/v1/tasks/stats` endpoint if one doesn't exist (return counts by status)

## Testing

- Open http://pi:5110/ and verify stats display correctly
- Check dark mode works
- Check mobile responsiveness
- Run `npx tsc --noEmit` and `npx eslint src/` from the frontend directory

## Safety

**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.** Task agents run inside Botka — deploying or restarting would kill the agent's own process. Only build and test.
