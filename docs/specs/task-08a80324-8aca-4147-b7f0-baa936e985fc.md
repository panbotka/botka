When Claude Code returns an error (like "Error in input stream"), show it inline in the chat as a collapsible error block (similar to tool call panels) instead of a red banner at the bottom. The raw error JSON should be visible on expand.

## Current behavior
- Error result events from Claude Code (result with `is_error: true`) are sent as SSE `event: error`
- Frontend catches this in `SSEContext.tsx` as `StreamError`, sets `session.state.streamError`
- `ChatView.tsx` shows it as a red banner at the bottom of chat (line ~881):
  ```
  {streamError && !isStreamingThisThread && (
    <div className="...bg-red-50 text-red-700...">
      <span className="flex-1 truncate">{streamError}</span>
      <button>Try again</button>
      <button>Dismiss</button>
    </div>
  )}
  ```
- The banner only shows the error text truncated, no details, and disappears on dismiss

## Expected behavior
- Error shows inline in the chat message list, like a red version of `ToolCallPanel`
- Collapsed: shows error icon + short error message (e.g. "Error in input stream")
- Expanded: shows raw error details (the full error JSON/text)
- "Try again" button remains available (either on the error block or next to it)
- The error block stays in the chat history — doesn't disappear like the banner
- Any partial response text that was streamed before the error should still be visible above the error block

## Requirements

### Frontend — Inline error component
- Create an inline error display similar to `ToolCallPanel` but styled for errors:
  - Red/rose border and background tint (not aggressive — `border-red-200 bg-red-50`)
  - Error icon (Lucide `AlertCircle` or `XCircle`)
  - Collapsed: one line showing error message
  - Expanded: raw error text/JSON in monospace, scrollable
  - "Retry" button on the collapsed header
- Component name suggestion: `StreamErrorBlock` or similar

### Frontend — Integration into chat flow
- When `streamError` is set on stream completion:
  - Instead of (or in addition to) setting the banner state, append the error as a visual element in the streaming message area
  - The error should appear below any partial response text that was already streamed
  - Keep the current banner ONLY for connection-level errors ("Server unavailable") where there's no chat context
- The inline error should include a "Retry" button that triggers `handleRegenerate()`

### Frontend — Error details
- For errors from Claude Code result events: the backend currently only sends `{"error": "message text"}` via SSE
- Enhance the SSE error event to include more context when available. In `chat.go` around line 552, when a result event has `is_error: true`, include the raw result JSON:
  ```go
  errJSON, _ := json.Marshal(map[string]interface{}{
    "error": event.ErrorMsg,
    "raw":   string(event.Raw),  // full result event JSON for debugging
  })
  ```
- Frontend parses `raw` from the error event and shows it in the expanded view

### Frontend — Styling
- Follow existing design language but use red/rose tones
- Collapsed state should look like a ToolCallPanel but with red accent
- Keep it compact — similar size to tool call panels
- The retry button should be clearly visible but not dominant

## Implementation Notes
- `ToolCallPanel.tsx` is a good reference for the collapsible pattern with the grid-rows transition
- The `streamError` state in `ChatView.tsx` currently controls the banner — rework this to feed the inline component instead
- In `SSEContext.tsx` line ~264, the error chunk could carry additional fields (`raw`) if the backend provides them
- The `StreamChunk` type in `client.ts` already has `error?: string` — extend to `error_raw?: string` for the expanded details
- Backend change is minimal: just add `event.Raw` to the error SSE event in `chat.go`
- Existing tests must pass (`make check`)
