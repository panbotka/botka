# Add Queue and Cancel Buttons to Pending Task Detail

When viewing a task with `pending` status on the task detail page, add action buttons to queue or cancel the task directly from the detail view.

## Requirements

- On the task detail page (`frontend/src/pages/TaskDetailPage.tsx`), when the task status is `pending`, show two action buttons:
  - **Queue** button — sets status to `queued` (uses `updateTask` API with `{ status: 'queued' }`)
  - **Cancel** button — sets status to `cancelled` (uses `updateTask` API with `{ status: 'cancelled' }`)
- The Queue button should be styled as a primary action (blue, similar to the existing Retry button style)
- The Cancel button should be styled as a destructive action (red, similar to the existing Delete button style)
- Use appropriate Lucide icons: `Play` or `ListPlus` for Queue, `XCircle` or `Ban` for Cancel
- After a successful status change, the task detail should refresh to reflect the new status and the buttons should update accordingly
- Place these buttons in the existing action buttons area alongside the Edit and Delete buttons

## Context

The backend already supports these transitions. The `ValidTransitions` map in `internal/models/task.go` defines `pending → queued` and `pending → cancelled`. The `updateTask` API endpoint (`PUT /api/v1/tasks/:id`) handles status changes. The frontend API client already has `updateTask(id, data)` in `frontend/src/api/client.ts`. No backend changes needed — this is frontend-only.

## Existing Pattern

Look at how other status-specific buttons are implemented in `TaskDetailPage.tsx`:
- Running tasks show a "Kill Task" button
- Failed/needs_review tasks show a "Retry" button
- needs_review tasks show a "Mark Done" button

Follow the same conditional rendering pattern: `{task.status === 'pending' && (...buttons...)}`

## Edge Cases

- If the API call fails, show an error (the existing error handling pattern in the component should be followed)
- The `cancelled` status already has a button condition for showing action buttons — make sure the cancelled task does NOT show Queue/Cancel buttons (only pending should)
