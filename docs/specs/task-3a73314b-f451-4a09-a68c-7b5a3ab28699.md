# Fix Command Route Registration Bug

## Problem

In `internal/handlers/command.go`, the `RegisterCommandRoutes` function has incorrect route paths for two of three endpoints. Lines 57-58 are missing a `/` before `:id`:

```go
rg.GET("/projects:id/commands", h.ListCommands)          // WRONG
rg.DELETE("/projects:id/commands/:pid", h.KillCommand)    // WRONG
```

Should be:

```go
rg.GET("/projects/:id/commands", h.ListCommands)          // CORRECT
rg.DELETE("/projects/:id/commands/:pid", h.KillCommand)    // CORRECT
```

Line 56 (`POST /projects/:id/run-command`) is correct, so running a command works but listing running commands and killing them does not — the Gin router never matches these malformed paths.

## Requirements

- Fix the two route paths in `RegisterCommandRoutes` in `internal/handlers/command.go` (add the missing `/` before `:id`)
- Run `make check` to ensure everything passes
- No other changes needed — the handler logic, tests, and frontend are all correct
