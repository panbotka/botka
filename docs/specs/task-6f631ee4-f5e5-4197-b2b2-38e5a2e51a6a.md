The cost numbers shown on the Dashboard (homepage) and the Cost tab (Cost Dashboard) don't match. They should show the same data.

## Requirements

### Investigate the discrepancy
- Read both components — the Dashboard page and the Cost Dashboard page
- Identify where each gets its cost data:
  - Are they calling different API endpoints?
  - Are they computing/aggregating differently (different time windows, different metrics)?
  - Is one using cached data and the other fresh?
  - Is one showing API cost and the other task cost?
- Find the root cause of the mismatch

### Fix
- Both should show the same cost numbers for the same time period
- If they show different granularity (e.g. Dashboard shows "today" summary, Cost tab shows detailed breakdown), the totals must still agree
- Use the same data source — ideally one API endpoint that both consume, or at minimum the same calculation logic
- If the Dashboard shows a summary card with cost, it should be derived from the same data as the Cost tab

### Verify
- After the fix, the numbers on Dashboard and Cost tab must match
- Test with current data — check that the displayed values are consistent

## Implementation Notes
- The cost data likely comes from the `claude-usage` command or from task cost tracking in the DB
- Check the API endpoints used by each page (look at the frontend API calls)
- The mismatch might be a simple unit issue (dollars vs cents), time window issue (today vs rolling 24h), or aggregation issue
- Existing tests must pass (`make check`)

## Safety
**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.**
