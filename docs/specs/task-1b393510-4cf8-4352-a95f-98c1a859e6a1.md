# Fix project picker double-unwrapping API response

The ProjectPicker in chat shows "No projects available" because `loadProjects` in ChatPage double-unwraps the API response envelope, resulting in projects always being an empty array.

## Requirements

- Fix `loadProjects` in `frontend/src/pages/ChatPage.tsx` (lines 104-114): `fetchProjects()` already returns `Project[]` via `requestData()` which unwraps `{ data: T }`, but `loadProjects` destructures `{ data }` again, getting `undefined`
- Replace the current double-unwrap logic with a simple call: fetch projects and set them directly
- Remove the redundant nested try/catch fallback (lines 108-112) which was a workaround attempt for the same bug
- After the fix, the ProjectPicker dropdown in the chat header should list all active projects
- Do NOT change `fetchProjects()` in `frontend/src/api/client.ts` — it is correct

## Implementation Notes

- The fix is in `frontend/src/pages/ChatPage.tsx` in the `loadProjects` callback
- Compare with `loadTags` (line 96-98) which correctly does `setTags(await api.fetchTags())` — follow the same pattern
