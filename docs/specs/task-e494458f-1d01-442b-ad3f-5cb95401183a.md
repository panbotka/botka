# Fix Live Output Showing "Done" for Newly Started Tasks

## Problem

Race condition: when a task starts, `claimTask()` sets status to `running` in DB before `launchTask()` creates the output buffer. The frontend sees `running`, connects to SSE `/api/v1/tasks/:id/output`, but the backend finds no buffer and immediately sends `event: done`. The frontend shows "DONE" even though the task just started.

## Root Cause

In `internal/runner/runner.go`:
1. `claimTask()` (line 413) sets `status = running` in DB and commits
2. `launchTask()` (line 461) creates the buffer — but this happens after the DB commit

In `internal/handlers/output.go` (line 43-51):
- `GetBuffer()` returns nil → sends `event: done` immediately

## Fix

Already partially applied in `output.go` — the SSE handler now polls for the buffer up to 10 times with 500ms intervals (5s total) before giving up with `done`. 

Verify the fix compiles and works:

```go
var buf *runner.Buffer
for range 10 {
    buf = r.GetBuffer(taskID)
    if buf != nil {
        break
    }
    time.Sleep(500 * time.Millisecond)
}
```
