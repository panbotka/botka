# Fix "bufio.Scanner: token too long" error in Claude stream reader

Chat sessions fail when Claude emits a single NDJSON event larger than 1MB (large tool_result, file read, etc.). The current `bufio.Scanner` with a 1MB buffer rejects such lines.

## Requirements

- Replace `bufio.Scanner` used for reading Claude stdout with `bufio.Reader.ReadBytes('\n')` (or equivalent) so NDJSON lines of any size are accepted.
- Apply the fix in both stdout readers:
  - `internal/claude/runner.go` (around line 215) — chat mode reader
  - `internal/claude/pool.go` (around line 228) — session pool reader
- Preserve current behavior: skip empty lines, pass each line to `parseEvent`, forward events on the channel, break on `KindResult` in pool.
- Handle EOF cleanly (no error emitted on normal end-of-stream).
- Stderr scanners can stay as `bufio.Scanner` but bump the buffer to 16MB for safety.
- Add or update a unit test that feeds a >1MB NDJSON line through the reader and verifies it is parsed without error.
- `make check` must pass.

## Implementation Notes

- `bufio.Reader.ReadBytes('\n')` returns the delimiter; trim it before calling `parseEvent`.
- On `io.EOF` with a non-empty trailing line, still parse it (matches `bufio.Scanner` semantics).
- Do not change the event channel contract or the `StreamEvent` shape.
