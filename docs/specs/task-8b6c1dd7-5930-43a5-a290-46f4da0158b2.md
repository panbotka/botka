## Goal

When a Claude Code session is actively running (streaming a response) for a thread, visually highlight that thread in the sidebar thread list. If the user is currently viewing that thread, the highlight should be even more prominent.

## Current State

- The sidebar shows a list of threads (`ThreadList` component).
- The SSE context (`SSEContext.tsx`) tracks which threads have active streaming sessions via `SSEManager`.
- The `ChatView` component already knows `isStreamingThisThread` for the currently viewed thread.
- There is no visual indication in the sidebar that a thread has an active session.

## Requirements

1. **Active session indicator in sidebar**: Threads with an actively streaming Claude session should have a visual indicator (e.g., a pulsing dot, animated border, or subtle glow/highlight).
2. **Current thread + active**: If the user is viewing the thread that's actively streaming, the sidebar entry should have an enhanced highlight (combining the "selected" and "active session" states).
3. **Non-current thread + active**: If a session is active on a thread the user is NOT currently viewing (e.g., queued messages still processing), it should still show the active indicator so the user knows something is happening there.

## Implementation Hints

- `SSEManager` has a `getSessionState(threadId)` method and tracks `isStreaming` per session.
- Expose a hook or method to check if a given threadId has an active streaming session (e.g., `sseManager.isStreaming(threadId)` already exists).
- In the thread list component, for each thread entry, check if it has an active session and apply appropriate CSS classes.
- Use Tailwind CSS for styling — e.g., `animate-pulse` on a small dot, or a left border accent, or a subtle background color change.
- Keep it subtle and tasteful — don't make the entire row flash. A small pulsing indicator or a colored left border is enough.

## Testing

- Run `make check` to ensure all linting and type checks pass.
- Verify the indicator appears when streaming and disappears when the stream completes.
