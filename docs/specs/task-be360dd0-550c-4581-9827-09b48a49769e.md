Replace the current favicon with a custom image and ensure it works across all platforms: browsers, PWA on iOS (Apple touch icon), and PWA on Android (manifest icons).

## Source Image

Download this image and use it as the source for generating all favicon sizes:

```
https://venngage-wordpress.s3.amazonaws.com/uploads/2022/09/meme_this_is_fine_dog.png
```

Save the original to `frontend/public/source-favicon.png` for reference.

## Required Files

Generate these files in `frontend/public/`:

1. **`favicon.ico`** — 32x32 ICO format (or multi-size 16x16 + 32x32)
2. **`favicon-16x16.png`** — 16x16 PNG
3. **`favicon-32x32.png`** — 32x32 PNG
4. **`apple-touch-icon.png`** — 180x180 PNG (iOS home screen icon — MUST be exactly 180x180)
5. **`android-chrome-192x192.png`** — 192x192 PNG (Android PWA icon)
6. **`android-chrome-512x512.png`** — 512x512 PNG (Android PWA splash/icon)
7. **`maskable-icon-512x512.png`** — 512x512 PNG with safe zone padding for Android maskable icon (the image should be inset ~10% from edges with a solid background color filling the rest)

Use ImageMagick (`convert` or `magick`) to resize. It's available on this system. Example commands:

```bash
# Download source
curl -L -o frontend/public/source-favicon.png "https://venngage-wordpress.s3.amazonaws.com/uploads/2022/09/meme_this_is_fine_dog.png"

# Generate sizes
convert frontend/public/source-favicon.png -resize 16x16 frontend/public/favicon-16x16.png
convert frontend/public/source-favicon.png -resize 32x32 frontend/public/favicon-32x32.png
convert frontend/public/source-favicon.png -resize 180x180 frontend/public/apple-touch-icon.png
convert frontend/public/source-favicon.png -resize 192x192 frontend/public/android-chrome-192x192.png
convert frontend/public/source-favicon.png -resize 512x512 frontend/public/android-chrome-512x512.png

# ICO with multiple sizes
convert frontend/public/source-favicon.png -resize 32x32 frontend/public/favicon.ico

# Maskable icon (with padding for safe zone)
convert frontend/public/source-favicon.png -resize 410x410 -gravity center -extent 512x512 -background white frontend/public/maskable-icon-512x512.png
```

Note: The source image may not be square. If not, crop it to square (center crop) before resizing, or resize with `-gravity center -extent WxH` to add padding.

## Update HTML (`frontend/index.html`)

Add/update favicon link tags in the `<head>`:

```html
<link rel="icon" type="image/x-icon" href="/favicon.ico" />
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png" />
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png" />
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png" />
```

Remove any existing favicon link tags that point to old icons.

## Update PWA Manifest

The PWA manifest is managed by `vite-plugin-pwa`. Find the manifest configuration in `frontend/vite.config.ts` (look for the `VitePWA` plugin config) and update the `icons` array:

```javascript
icons: [
  {
    src: '/android-chrome-192x192.png',
    sizes: '192x192',
    type: 'image/png',
  },
  {
    src: '/android-chrome-512x512.png',
    sizes: '512x512',
    type: 'image/png',
  },
  {
    src: '/maskable-icon-512x512.png',
    sizes: '512x512',
    type: 'image/png',
    purpose: 'maskable',
  },
]
```

Also ensure the manifest has `theme_color` and `background_color` set (they likely already are).

## Verification

1. Run `cd frontend && npm run build` — must compile without errors
2. Check all generated PNG files exist and have correct dimensions: `identify frontend/public/*.png`
3. Check favicon.ico exists: `identify frontend/public/favicon.ico`
4. The built output should include all icon files in `frontend/dist/`
5. Remove `frontend/public/source-favicon.png` after generating all icons (don't ship the large source file)
