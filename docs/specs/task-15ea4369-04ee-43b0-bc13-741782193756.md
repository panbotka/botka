The browser tab title only updates on the Chat page. All other pages (Tasks, Projects, Dashboard, Settings, Help, Task Detail) keep showing whatever title was last set. Each page should set `document.title` appropriately.

## Current State

Only `frontend/src/pages/ChatPage.tsx:78-84` sets the title:

```tsx
useEffect(() => {
  document.title = activeThread?.title
    ? `${activeThread.title} — Botka`
    : 'Botka'
}, [activeThread?.title, activeThreadId])
```

No other page sets `document.title`.

## Fix

### Option A: Hook (preferred)

Create a simple `useDocumentTitle` hook in `frontend/src/hooks/useDocumentTitle.ts`:

```typescript
import { useEffect } from 'react'

export function useDocumentTitle(title: string) {
  useEffect(() => {
    document.title = title ? `${title} — Botka` : 'Botka'
  }, [title])
}
```

### Option B: Set title directly in each page via useEffect

Either way, add title setting to each page:

### Integration with notifications

The notification hook in `frontend/src/hooks/useNotifications.ts:22-23` reads the base title via `getBaseTitle()` which strips the `(N)` unread prefix. The `— Botka` suffix pattern is consistent with this — just make sure the title format stays as `{Page} — Botka` so the notification prefix `(N) {Page} — Botka` works correctly.
