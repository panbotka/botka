## Goal

Add a new tab/page in the Botka sidebar menu that shows the status of the "Box" desktop server (AMD Ryzen 9 + RTX 3070, Ubuntu 24.04, hostname `box`). The box is a secondary server that runs intermittently — the user needs to see if it's online, start/stop it, and see which services are running.

## Box Details

- **Hostname:** box
- **LAN IP:** 192.168.10.101
- **Tailscale IP:** 100.127.79.1
- **SSH:** `ssh box@box` (via Tailscale)
- **Wake-on-LAN script:** `/home/pi/bin/boxon` (runs from the RPi)
- **MAC:** 3C-7C-3F-1C-E6-B3

### Services on Box

| Service | Port | Type | Description |
|---------|------|------|-------------|
| image-embeddings | 8000 | systemd | OpenCLIP + InsightFace API (GPU, ~1.9 GB VRAM) |
| photo-enhancer | 8001 | systemd | AI photo processing (GPU, ~3.4 GB VRAM) |
| llama.cpp | 8080 | manual | LLM inference (when running) |

## Requirements

### 1. Backend — Box status API

Create a new handler (`internal/handlers/box.go`) with endpoints:

- **`GET /api/v1/box/status`** — Check if the box is online (ping or SSH check via Tailscale IP `100.127.79.1`). Return:
  - `online: bool`
  - `services: [{name, port, status (running/stopped), url}]`
  - For each known service, check if the port is reachable (e.g., TCP connect to `100.127.79.1:<port>` with short timeout)

- **`POST /api/v1/box/wake`** — Run `/home/pi/bin/boxon` to send Wake-on-LAN magic packet. Return success/failure.

- **`POST /api/v1/box/shutdown`** — Run `ssh box@box sudo shutdown now` to shut down the box. Return success/failure.

- **`POST /api/v1/box/services/:name/start`** — Run `ssh box@box sudo systemctl start <name>` to start a service.

- **`POST /api/v1/box/services/:name/stop`** — Run `ssh box@box sudo systemctl stop <name>` to stop a service.

**Security:** Whitelist allowed service names (`image-embeddings`, `photo-enhancer`) to prevent command injection. Do NOT allow arbitrary service names. llama.cpp is manual and should not have start/stop controls (or use a different mechanism if it has a script).

### 2. Frontend — Box dashboard page

Create a new page (`frontend/src/pages/BoxPage.tsx`) accessible from the sidebar navigation.

**Layout:**
- **Status header:** Large indicator showing if box is online (green) or offline (red/gray). Show hostname and IP.
- **Power controls:** "Wake" button (disabled when online), "Shutdown" button (disabled when offline, with confirmation dialog).
- **Services list:** Card for each service showing:
  - Name and description
  - Status badge (running/stopped)
  - Port number
  - Link to the service URL (`http://100.127.79.1:<port>`) — clickable when running
  - Start/Stop button (for systemd services)
  - VRAM usage info (static, from config)

**Auto-refresh:** Poll `/api/v1/box/status` every 10 seconds when the page is visible. After wake/shutdown actions, poll more frequently (every 3s) for 60 seconds to catch state changes.

### 3. Sidebar navigation

Add a "Box" entry to the sidebar nav (use the `Server` or `Monitor` icon from Lucide). Place it after existing nav items.

### 4. Configuration

Add environment variables:
- `BOX_HOST` (default: `100.127.79.1`) — Tailscale IP for SSH and health checks
- `BOX_SSH_USER` (default: `box`) — SSH username
- `BOX_WOL_COMMAND` (default: `/home/pi/bin/boxon`) — Wake-on-LAN command path

Store service definitions in config (not hardcoded in frontend):
```go
type BoxService struct {
    Name        string
    Port        int
    Description string
    Type        string // "systemd" or "manual"
    VRAMUsageMB int    // optional, for display
}
```

## Implementation Hints

- Existing nav/sidebar is in `frontend/src/components/ThreadSidebar.tsx` or a layout component — check where the current nav tabs (Chat, Tasks, etc.) are rendered.
- For TCP port checks, use Go's `net.DialTimeout` with a 2-second timeout.
- For SSH commands, use `exec.CommandContext` with a timeout (10s for status checks, 30s for start/stop).
- Follow existing patterns: handler registration in `cmd/server/`, API client in `frontend/src/api/client.ts`, types in `frontend/src/types/index.ts`.
