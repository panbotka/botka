# Fix Chat Auto-Scroll Behavior (Stick to Bottom)

The chat view should auto-scroll to follow new content only when the user is already at the bottom of the conversation. If the user has scrolled up to read older messages, new content (streaming text, tool calls, tool results) should NOT force-scroll them down.

## Current Behavior

In `frontend/src/components/ChatView.tsx` (around line 340-371), the auto-scroll logic uses a `distanceFromBottom < 150` threshold check on every render triggered by `messages`, `streamingContent`, `streamingThinking`, or `threadId` changes. This approach has issues:

1. The 150px threshold is fragile — new content can push the scroll position beyond it before the check runs, causing the user to "unstick" from the bottom unintentionally
2. `scrollIntoView()` is called without `behavior: 'smooth'`, causing jarring jumps
3. There's no explicit tracking of whether the user intentionally scrolled away

## Requirements

### Implement "Stick to Bottom" State Tracking

- Track an `isAtBottom` boolean state using a scroll event listener on the scroll container (`scrollContainerRef`)
- The user is "at bottom" when `scrollHeight - scrollTop - clientHeight < threshold` (use ~50px threshold to account for sub-pixel rounding)
- Update `isAtBottom` on every scroll event (use passive listener for performance)
- When the user scrolls up beyond the threshold, set `isAtBottom = false` — this means "user is reading history, don't auto-scroll"
- When the user scrolls back down to the bottom, set `isAtBottom = true` — resume auto-scrolling

### Auto-Scroll Rules

- **User sends a message:** Always scroll to bottom (existing `userSentRef` logic is correct, keep it)
- **Thread switch:** Always scroll to bottom (existing logic is correct, keep it)
- **New streaming content arrives** (AI response text, tool calls, tool results): Only scroll to bottom if `isAtBottom` is true
- **Messages array changes** (new message added after stream completes): Only scroll to bottom if `isAtBottom` is true
- Use `scrollIntoView({ behavior: 'smooth' })` for auto-scroll during streaming, but keep instant scroll for thread switches and user-sent messages

### Edge Cases

- When a stream starts (first chunk of AI response), if the user was at the bottom, they should stay at the bottom as content grows
- Rapid scroll events should not cause performance issues — debounce or throttle the scroll handler if needed, or use a simple ref without triggering re-renders
- The `isAtBottom` ref should NOT be React state (no `useState`) — it should be a `useRef` to avoid unnecessary re-renders on every scroll event
- Window resize should re-evaluate `isAtBottom` (the container height changes)

## Implementation Notes

- The scroll container is `scrollContainerRef` (the `div` with `className="flex-1 overflow-y-auto"`)
- The bottom sentinel is `bottomRef`
- Keep the existing thread-switch and user-sent-message scroll logic intact
- Replace the current `distanceFromBottom < 150` check with the `isAtBottom` ref check
