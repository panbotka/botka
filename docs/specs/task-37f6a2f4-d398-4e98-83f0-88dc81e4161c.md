## Problem

The `ProcessBar` component (`frontend/src/components/ProcessBar.tsx`) disappears when there are no active processes (`processes.length === 0` → returns `null`). This means the bar keeps flickering or vanishing while Botka is actively responding, because the `/processes` polling (every 3s in `useProcesses` hook) may temporarily return an empty list between process states.

The user wants the active sessions bar to always be visible — not just when processes are running.

## Requirements

1. **Always show the ProcessBar** — never return `null`. The bar should be visible at all times, even when no processes are running.
2. When no processes are active, show an empty/idle state (e.g. "No active sessions" or just the bar with no session badges).
3. Keep the existing behavior when processes ARE running (green pulsing dot, session badges with duration and kill button).

## Files to modify

- `frontend/src/components/ProcessBar.tsx` — Remove the early `null` return when `processes.length === 0`. Show an idle state instead.

## Implementation notes

- Keep the bar visually minimal when idle — don't make it distracting. A subtle bar with muted text like "No active sessions" or just the "Active:" label with no badges is fine.
- The bar renders in two places in `ChatPage.tsx` (mobile + desktop) — both should behave consistently.
- Run `make check` to verify nothing breaks.
