# Change MCP create_task Default Status from Queued to Pending

The MCP `create_task` tool defaults to `queued` when no status is provided. It should default to `pending` so tasks are created as drafts that the user explicitly queues when ready.

## Root Cause

In `internal/mcp/tools.go` line 47, the default status is hardcoded to `queued`:

```go
status := models.TaskStatusQueued
```

## Requirements

- Change the default in `internal/mcp/tools.go` line 47 from `models.TaskStatusQueued` to `models.TaskStatusPending`
- Update the MCP tool schema description in `internal/mcp/server.go` (the `enumProp` call for `status` on the `create_task` tool) to reflect that `pending` is the default
- Update any tests in `internal/mcp/tools_test.go` that rely on the default being `queued`

## Files to Change

- `internal/mcp/tools.go` — change default on line 47
- `internal/mcp/server.go` — update schema description for the status field
- `internal/mcp/tools_test.go` — update tests expecting default `queued`
