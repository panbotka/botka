# Cost dashboard with daily/weekly spend charts

Add a cost analytics dashboard showing API spend over time, broken down by thread and project.

## Requirements

### Backend — Cost analytics endpoint
- Add `GET /api/v1/analytics/cost` endpoint with query params:
  - `period`: `day`, `week`, `month` (default: `week`)
  - `days`: number of days to look back (default: 30)
- Response: `{"data": {"total_cost_usd": N, "by_date": [{"date": "2026-03-25", "cost_usd": N, "input_tokens": N, "output_tokens": N}], "by_thread": [{"thread_id": N, "title": "...", "cost_usd": N}], "by_project": [{"project_name": "...", "cost_usd": N}]}}`
- Aggregate data from both chat messages (`messages.cost_usd`) and task executions (`task_executions.cost_usd`)
- `by_thread` and `by_project` return top 10 sorted by cost descending

### Frontend — Dashboard page
- Add a "Cost" tab or section accessible from the main navigation
- Show:
  - Total spend for selected period (big number at top)
  - Bar chart or area chart: cost per day over the selected period
  - Top threads by cost (list with bar visualization)
  - Top projects by cost (list with bar visualization)
- Period selector: 7 days / 30 days / 90 days
- Use simple CSS/SVG bars or a lightweight chart approach — no heavy chart library needed

### Frontend — Styling
- Follow existing app design language (Tailwind, zinc palette)
- Clean, minimal dashboard layout
- Numbers formatted as currency: "$1.23"

## Implementation Notes

- This task depends on "Track and display cost per thread" being done first (needs `cost_usd` on messages)
- For the chart, simple CSS bars (div with width percentage) work fine — no need for Chart.js or similar
- Task execution costs are already stored in `task_executions.cost_usd`
- Existing tests must pass (`make check`)

## Safety

**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.**
