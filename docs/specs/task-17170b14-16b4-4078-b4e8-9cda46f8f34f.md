The API usage tab (Cost Dashboard) should auto-refresh data every time the user navigates to it. Also, the "last refreshed" time should be a live ticking counter, not a static timestamp.

## Requirements

### Auto-refresh on tab visit
- Every time the user navigates to the Cost Dashboard page, trigger a fresh data fetch
- This is a cheap operation (just calls `claude-usage` command) so no throttling needed
- Don't show stale cached data — always fetch fresh on mount
- Also refresh when the browser tab becomes visible again (Page Visibility API: `document.addEventListener('visibilitychange', ...)`)

### Live elapsed timer since last refresh
- Instead of showing a static timestamp like "Last updated: 14:32", show a live ticking counter: "Updated 5s ago", "Updated 1m 23s ago", etc.
- The counter should tick every second, updating in real-time
- Format:
  - Under 60s: "Xs ago" (e.g. "12s ago")
  - 1-59 minutes: "Xm Ys ago" (e.g. "3m 45s ago")
  - 60+ minutes: "Xh Ym ago" (e.g. "1h 15m ago")
- Reset to "0s ago" when data is refreshed
- Use `useEffect` with `setInterval(1000)` for the ticking, clean up on unmount

### Implementation
- The refresh function likely already exists — just call it on component mount and on visibility change
- Store `lastRefreshedAt` as a `Date` object in state
- Create a small hook or inline effect that computes the elapsed time every second
- The timer display should be subtle (small muted text near the refresh button or header)

## Implementation Notes
- Changes are in the Cost Dashboard page component
- If there's a manual "Refresh" button, keep it — clicking it also resets the timer
- Don't use `react-query` or similar — keep it simple with `useState`/`useEffect`
- The visibility change listener should be added/removed in a `useEffect` cleanup
- Existing tests must pass (`make check`)
