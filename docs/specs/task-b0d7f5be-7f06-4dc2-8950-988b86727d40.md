## Bug

Když se stav tasku automaticky změní (např. queued → running → done/failed), tabulka na stránce Tasks to neaktualizuje v reálném čase. Hlavně počty v názvech tabů (např. "Queued (11)", "Running (1)") zůstávají zastaralé.

## Očekávané chování

Když runner změní stav tasku, UI by mělo okamžitě (nebo s minimálním zpožděním) reflektovat:
1. Přesun tasku mezi taby (např. z Queued do Running)
2. Aktualizované počty v názvech tabů

## Kde hledat

1. **Frontend:** Stránka Tasks — `frontend/src/pages/` a komponenty pro task tabulku/taby. Hledej jak se refreshují data po změně stavu.
2. **SSE events:** Zkontroluj jestli backend posílá SSE event při změně stavu tasku a jestli frontend na něj reaguje reloadem dat. Hledej `task` related SSE events v `internal/handlers/` a `frontend/src/hooks/` nebo `frontend/src/context/SSEContext`.
3. **Polling:** Pokud se nepoužívá SSE pro task updates, možná chybí polling nebo je interval příliš dlouhý.

## Řešení

Nejspíš je potřeba:
- Zajistit že runner posílá SSE event při každé změně stavu tasku
- Frontend musí na tento event reagovat refreshem task listu včetně počtů v tabech
- Případně přidat polling jako fallback

## Reprodukce

1. Otevři Tasks stránku
2. Měj queued tasky které runner postupně zpracovává
3. Pozoruj že počty v tabech a pozice tasků v tabulce se neaktualizují automaticky
