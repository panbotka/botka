# Move API Usage Section to Top of Dashboard

The API Usage meters are currently near the bottom of the dashboard page (7th section out of 8). They should be the very first section after the page title because they are the most frequently checked information.

## Requirements

In `frontend/src/pages/DashboardPage.tsx`, move the `<UsageMeters>` component rendering (currently around line 616-619) to immediately after the page title `<h1>Dashboard</h1>`, before the Runner Controls section.

The current section order is:
1. Page title
2. Runner Controls
3. Active Chat Sessions
4. Task Status Grid #1 (running, queued, done, failed)
5. Task Status Grid #2 (needs review, pending, total, success rate)
6. Extra Insights Row (avg duration, total cost, most active project)
7. **API Usage (UsageMeters)** ← move this
8. Cost/Token Usage (CostContent)

New order should be:
1. Page title
2. **API Usage (UsageMeters)** ← here
3. Runner Controls
4. Active Chat Sessions
5. Task Status Grid #1
6. Task Status Grid #2
7. Extra Insights Row
8. Cost/Token Usage (CostContent)

This is a simple cut-and-paste of the JSX block. No logic changes needed — the component already receives its props (`usage` from `runnerStatus.usage` and `onRefresh` callback) which don't depend on render position.
