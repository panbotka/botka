# Fix Live Output Not Visible on Task Detail Page

When viewing a task's detail page during execution, the live streaming output from the Claude Code session is not visible.

## Root Cause

The `TaskDetailPage` component does not subscribe to task status change events. When a user is viewing a `queued` task and the runner picks it up (changing status to `running`), the page doesn't know the status changed. Since `LiveOutputInline` only renders when `task.status === 'running'`, the live output component never appears.

The task list page (`TasksPage.tsx`) already uses `useTaskEvents(refetchAll)` to auto-refresh on status changes, but `TaskDetailPage.tsx` does not.

## Requirements

### Primary Fix: Auto-refresh task detail on status changes

In `frontend/src/pages/TaskDetailPage.tsx`, add `useTaskEvents` (from `../hooks/useTaskEvents`) to the `TaskDetail` component so it calls `load()` whenever a task status event is received. This is identical to how `TasksPage.tsx` uses it at line 101: `useTaskEvents(refetchAll)`.

The hook connects to the SSE endpoint `/api/v1/tasks/events` and fires the callback on every `task_status` event. Adding `useTaskEvents(load)` to `TaskDetail` will cause the page to re-fetch task data whenever any task changes status, which will trigger the `LiveOutputInline` component to render when the task transitions to `running`.

### Secondary Fix: Add polling fallback for running tasks

As a safety net, add polling that refreshes task data every 5 seconds while the task status is `running`. This handles edge cases where the SSE event connection drops. Use `setInterval` inside a `useEffect` that depends on `task?.status`, and clear it when status is no longer `running`.

## Files to Change

- `frontend/src/pages/TaskDetailPage.tsx` — the only file that needs changes

## Verification

1. Navigate to a task detail page while the task is `queued`
2. When the runner picks it up and changes to `running`, the page should automatically update to show the running status badge and the live output panel
3. The live output should stream in real-time showing Claude Code's activity
4. When the task completes, the page should update to show the final status and historical output
