## Task 4: Project Discovery and API Handler

Port Saiduler's project discovery and project API handler to Botka.

### Context
Projects are the unified entity replacing both Saiduler's "projects" and Chatovadlo's "folders". They represent filesystem directories (typically git repos) that can be used for both task scheduling and chat workspaces.

### Reference Files
- `/home/pi/projects/saiduler/internal/projects/discovery.go` — Scan + SyncToDatabase functions
- `/home/pi/projects/saiduler/internal/handlers/project.go` — Project HTTP handlers
- `/home/pi/projects/saiduler/internal/handlers/response.go` — respondOK/respondError helpers

### What to Create

1. **`internal/handlers/response.go`** — Copy Saiduler's response helpers:
   - `respondOK(c *gin.Context, data interface{})` — wraps in `{"data": ...}`
   - `respondList(c *gin.Context, data interface{}, total int64)` — wraps in `{"data": ..., "total": N}`
   - `respondError(c *gin.Context, status int, msg string)`

2. **`internal/projects/discovery.go`** — Port from Saiduler:
   - `Scan(dir string) ([]DiscoveredProject, error)` — walks ProjectsDir, finds git repos
   - `SyncToDatabase(db *gorm.DB, discovered []DiscoveredProject) error` — upserts into projects table
   - The sync must preserve the new `claude_md` and `sort_order` columns (don't overwrite them on update)

3. **`internal/handlers/project.go`** — Port from Saiduler with extensions for the unified model:
   - `GET /api/v1/projects` — List active projects with task counts (and thread counts too)
   - `GET /api/v1/projects/:id` — Get single project
   - `PUT /api/v1/projects/:id` — Update project (branch_strategy, verification_command, claude_md, sort_order, active)
   - `POST /api/v1/projects/scan` — Trigger discovery scan

4. **Wire routes** in `cmd/server/main.go` setupRouter function.

5. **Update main.go run()** to call projects.Scan + SyncToDatabase on startup (like Saiduler does).

### Verification
- `GET /api/v1/projects` returns discovered projects from /home/pi/projects/
- `PUT /api/v1/projects/:id` can update claude_md and branch_strategy
- `POST /api/v1/projects/scan` discovers new repos
