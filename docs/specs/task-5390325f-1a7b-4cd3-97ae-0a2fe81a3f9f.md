## Problem

On the task detail page (`/tasks/:id`), long text inside the spec/description box overflows its container on the right side. For example, a line like "When creating tasks or discussing code, default to this project unless the user specifies otherwise." breaks out of the box.

This likely affects code blocks and other long unbroken content in the rendered markdown as well.

## File

`frontend/src/pages/TaskDetailPage.tsx` — find the container that renders the task spec/description.

## Fix

Add overflow handling CSS to the spec container:
- `overflow-wrap: break-word` (Tailwind: `break-words`)
- `overflow-x: auto` or `overflow-x: hidden` on the container
- For code blocks inside markdown: `overflow-x: auto` so they scroll horizontally instead of breaking out
- Make sure `max-width: 100%` or `min-width: 0` is set on flex children if inside a flex layout

Check if the spec is rendered as markdown — if so, also ensure the markdown renderer's container has proper overflow handling.

## Safety

**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.** Task agents run inside Botka — deploying or restarting would kill the agent's own process. Only build and test.
