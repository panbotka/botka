# Fix: pass uploaded files to Claude Code session

Uploaded files are saved to disk and DB but never referenced in the prompt sent to Claude. The agent has no way to know about attached files.

## Current behavior
1. User uploads a file via multipart form in `SendMessage`
2. File is saved to `data/uploads/<uuid>.ext` and an `attachments` DB record is created
3. The prompt sent to Claude is just the text `content` — no mention of the file
4. If content is empty, it sends "(attached files)" which is useless without paths

## Expected behavior
- Claude receives the file path(s) in the prompt so it can read them with its `Read` tool
- For images: Claude Code's `Read` tool can display images natively — just provide the absolute path
- For text/PDF: same — `Read` tool handles these

## Requirements

### Prompt enrichment
- After saving uploaded files, append file references to the prompt sent to Claude
- Format: append a section like:
  ```
  
  Attached files:
  - /absolute/path/to/uploads/uuid.png (image/png, original: screenshot.png)
  - /absolute/path/to/uploads/uuid.pdf (application/pdf, original: report.pdf)
  ```
- Use absolute paths so Claude can read them regardless of working directory
- Include original filename and MIME type for context
- For images, add a hint: "Use the Read tool to view image files."

### Edge cases
- Multiple files: list all of them
- Text-only message with no files: no change to prompt
- Files-only (no text): replace "(attached files)" with proper file listing

## Implementation Notes

- Changes are in `chat.go` `SendMessage`, between file saving (line ~140) and `streamResponse` call (line ~151)
- The `saveUploadedFile` returns `*models.Attachment` which has `StoredName` and `OriginalName` — use these to build the path
- Upload dir path is `h.uploadDir` — convert to absolute path if relative
- Existing tests must pass (`make check`)

## Safety

**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.**
