# Fix AskUserQuestion Tool Not Working

The `AskUserQuestion` tool from the superpowers skill fails with `"Answer questions?"` error when invoked during a Botka-managed Claude Code session (via OpenClaw gateway). The tool should either work properly or be gracefully handled.

## Context

When Claude Code runs via the OpenClaw gateway (port 18789) and tries to use the `AskUserQuestion` tool (from the superpowers skills), it fails with the error message `"Answer questions?"`. This happens because the tool requires interactive user input through the Claude Code UI, but Botka runs sessions non-interactively (`--dangerously-skip-permissions`).

This was observed during a product-manager skill session where Claude tried to interview the user about feature requirements.

## Requirements

- Investigate how AskUserQuestion is invoked and why it fails in non-interactive mode
- Determine the root cause: is it the OpenClaw gateway, the Claude Code SDK, or the tool implementation?
- Implement a fix. Possible approaches:
  1. **Detect non-interactive mode** — if running under Botka/non-interactive, skip AskUserQuestion calls gracefully (return a message like "Running in non-interactive mode, cannot ask questions")
  2. **Gateway-level handling** — if OpenClaw gateway intercepts tool calls, handle AskUserQuestion appropriately
  3. **Botka runner config** — pass a flag or env var that tells Claude Code sessions they're non-interactive

## Notes
- The error `"Answer questions?"` suggests the tool is partially working but the UI prompt component is returning an error or default response
- This is NOT about making interactive questions work in Botka sessions — those are by design non-interactive. The goal is graceful degradation so skills that use AskUserQuestion don't break entirely
- Check how other permission-dependent tools behave in `--dangerously-skip-permissions` mode for reference
