# Fix Cmd+K search result priority order

Search results in the global search overlay (Cmd+K) display in wrong priority order.

## Requirements

- Change the `toItems()` function in `frontend/src/components/SearchOverlay.tsx` to build items in this order: threads, tasks, projects, messages (currently: tasks, projects, threads, messages)
- Update the input placeholder text to match the new priority: "Search threads, tasks, projects, messages..."
- No backend changes needed — this is purely a frontend reordering of how results are grouped and displayed

## Acceptance Criteria

- When opening Cmd+K and searching, threads appear first, then tasks, then projects, then messages
- Placeholder text reads "Search threads, tasks, projects, messages..."
