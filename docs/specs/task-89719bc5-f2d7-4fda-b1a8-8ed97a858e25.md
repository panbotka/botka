# Prevent Task Agents From Self-Deploying Botka

## Problem

A task agent running on the botka project executed `make deploy`, which runs `systemctl restart botka`, killing itself and all other running tasks. This happened despite CLAUDE.md explicitly saying: "NEVER run `make deploy`, `make install-service`, `systemctl restart botka`..."

The CLAUDE.md warning works for interactive Claude sessions but task agents spawned by the scheduler might not always respect it (they can lose context or ignore instructions).

## Requirements

### 1. Add a blocked commands check in the executor

In `internal/runner/executor.go`, before spawning the Claude Code process for a task on the botka project, add an extra safeguard. The most reliable approach is to use Claude Code's `--disallowed-tools` flag or environment-level protection.

However, since the task agent runs arbitrary bash commands via Claude Code's tools, the real fix is:

### 2. Add a wrapper script that blocks dangerous commands

Create a script at `scripts/safe-make.sh` (or similar) that wraps `make` and blocks specific targets when running inside a task agent:

```bash
#!/bin/bash
# Block dangerous make targets when running as a Botka task agent
BLOCKED_TARGETS="deploy install-service"
for target in $BLOCKED_TARGETS; do
    if echo "$@" | grep -qw "$target"; then
        echo "ERROR: 'make $target' is blocked inside task agents to prevent self-restart"
        exit 1
    fi
done
exec make "$@"
```

### 3. Block systemctl restart/stop botka in task context

In `internal/runner/executor.go`, set an environment variable like `BOTKA_TASK_AGENT=1` when spawning Claude Code for task execution. Then add a check (via a shell alias, wrapper, or pre-exec hook) that prevents `systemctl restart botka` and `systemctl stop botka` when this env var is set.

The simplest approach: In the executor, when the project being executed IS the botka project (check by project name or path), add to the Claude Code command's `--disallowed-tools` or pass `--append-system-prompt` with an extra strong warning.

### 4. Strengthen CLAUDE.md warning

Update the "Task Agent Safety" section in CLAUDE.md to be even more explicit:

```
## Task Agent Safety

**CRITICAL: Running `make deploy`, `make install-service`, `systemctl restart botka`, 
or `systemctl stop botka` WILL KILL THIS PROCESS and leave the task stuck forever. 
This applies to ALL commands that restart the botka systemd service.**

If your task requires deploying changes, mark the task as done and note that 
deployment is needed — the user will deploy manually.
```

### 5. Add executor-level command filtering (belt and suspenders)

In `internal/runner/executor.go`, when building the Claude Code command for the botka project, add `--append-system-prompt` with content like:

```
CRITICAL SAFETY RULE: You are running as an autonomous task agent inside the Botka process. 
Running 'make deploy', 'make install-service', 'systemctl restart botka', or 'systemctl stop botka' 
will kill your own process immediately. NEVER run these commands. If deployment is needed, 
just commit your changes and note it in the task output.
```

This provides a runtime reminder that's harder to miss than a static CLAUDE.md file.
