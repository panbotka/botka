## Task 13: Settings Page

Create a unified Settings page combining configuration from both Saiduler and Chatovadlo.

### Context
The Settings page provides management for personas, tags, memories, and app preferences. It replaces placeholder SettingsPage from Task 3.

### Reference Files
- `/home/pi/projects/chatovadlo/frontend/src/components/SettingsPanel.tsx` — Chatovadlo's settings (theme, font, personas, tags, memories, folders)
- `/home/pi/projects/saiduler/frontend/src/pages/SettingsPage.tsx` — Saiduler's settings (if any)

### What to Create

**`pages/SettingsPage.tsx`** with tabbed sections:

1. **General** tab:
   - Theme toggle (light/dark) — saves to SettingsContext + localStorage
   - Font size selector (small/medium/large)
   - Default AI model selector
   - Notification sound toggle

2. **Personas** tab:
   - List of personas with icon, name, model
   - Create/edit persona form: name, icon (emoji picker), system prompt (textarea), default model, starter message, sort order
   - Delete persona with confirmation
   - Drag-to-reorder

3. **Tags** tab:
   - List of tags with color swatch and name
   - Create/edit: name, color (color picker)
   - Delete with confirmation
   - Show thread count per tag

4. **Memories** tab:
   - List of app-level memories
   - Create: text content
   - Edit inline
   - Delete with confirmation
   - Explanation text: "Memories are included in every chat session's system prompt"

5. **Voice** tab (if whisper enabled):
   - Transcription status indicator
   - Test microphone button

### Design
- Use Saiduler's design patterns: clean cards, zinc backgrounds, emerald accent buttons
- Tab navigation with underline indicator (like Saiduler's filter tabs)
- Forms match Saiduler's input styling

### Verification
- All tabs render and switch correctly
- CRUD operations for personas, tags, memories work
- Theme toggle switches between light and dark
- Settings persist across page reloads (localStorage)
