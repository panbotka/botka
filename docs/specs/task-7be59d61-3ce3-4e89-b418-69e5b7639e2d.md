## Task 14: PWA Optimization and Mobile Layout

Finalize PWA configuration and ensure the app works well on mobile devices.

### Context
The app must be optimized for PWA (Progressive Web App) — installable on mobile, offline-capable for static assets, responsive on all screen sizes. Chatovadlo already had PWA via vite-plugin-pwa; this task ensures it's properly configured for Botka.

### Reference Files
- `/home/pi/projects/chatovadlo/frontend/vite.config.ts` — PWA plugin config
- `/home/pi/projects/chatovadlo/frontend/src/components/BottomNav.tsx` — Mobile bottom nav
- `/home/pi/projects/chatovadlo/frontend/src/hooks/useIsMobile.ts` — Mobile detection

### What to Do

1. **PWA Manifest** — Verify vite-plugin-pwa config in vite.config.ts:
   - name: "Botka"
   - short_name: "Botka"  
   - theme_color: "#18181b" (zinc-900)
   - background_color: "#fafafa" (zinc-50)
   - display: "standalone"
   - orientation: "any"
   - Icons: generate proper 192x192 and 512x512 PNG icons with Botka branding (simple "B" letter on zinc background, or a robot emoji)

2. **Service Worker** — Verify workbox config:
   - Pre-cache all static assets (JS, CSS, HTML, fonts, icons)
   - navigateFallback to index.html for SPA routing
   - Exclude /api/ and /mcp/ from caching
   - Auto-update strategy (skipWaiting + clientsClaim)

3. **Mobile Responsive Layout**:
   - Sidebar collapses on screens < 768px
   - Chat page: single-panel mode on mobile (thread list OR chat, not both)
   - Bottom navigation bar on mobile with: Dashboard, Chat, Tasks, Settings icons
   - Touch-friendly tap targets (min 44px)
   - Swipe gestures for thread actions (if practical)
   - FAB (floating action button) for "new chat" on mobile chat view

4. **Viewport and Meta Tags** in index.html:
   - `<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">`
   - `<meta name="apple-mobile-web-app-capable" content="yes">`
   - `<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">`
   - `<link rel="apple-touch-icon" href="/icon-192.png">`

5. **Splash Screen** — Apple splash screen meta tags for iOS PWA

6. **Offline Indicator** — Show a subtle banner when offline ("No internet connection — some features may be unavailable")

### Verification
- `npm run build` generates service worker and manifest
- App is installable via Chrome "Add to Home Screen"
- Offline: static pages load, API calls show appropriate error
- Mobile: responsive layout works at 375px, 414px, 768px widths
- Bottom nav shows on mobile, sidebar hidden
- Touch targets are large enough
- No horizontal scroll on mobile
