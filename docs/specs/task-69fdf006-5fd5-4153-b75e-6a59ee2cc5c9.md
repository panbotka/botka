# Assistant File Attachment Support in Chat

When the assistant (Claude Code) produces, saves, or references files during a chat response, automatically attach them to the assistant message so they display inline in the chat UI.

## Supported File Types

- **Images:** `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`
- **Documents:** `.pdf`, `.html`, `.json`, `.csv`

## Requirements

### Backend: File Path Collection During Streaming

In `internal/claude/runner.go`, collect file paths from Claude Code NDJSON stream events during a chat response:

- **tool_use events:** Extract `file_path` parameter from Write tool calls. Extract file paths from Bash tool command strings and outputs.
- **tool_result events:** Scan content for absolute file paths ending in supported extensions.
- Store collected paths in a list associated with the current streaming session.

### Backend: Attachment Creation After Message Completion

In `internal/handlers/chat.go`, after the assistant message is saved to the database:

1. Iterate through collected file paths from the stream.
2. Filter for supported extensions (images, pdf, html, json, csv).
3. For each path that exists on disk and is a regular file:
   - Copy the file to the uploads directory (`UPLOAD_DIR`) with a generated stored name (same pattern as user uploads).
   - Create an `Attachment` record linked to the assistant message with correct `mime_type`, `original_name`, `stored_name`, and `size`.
4. If any attachments were created, send them to the frontend via SSE so they appear without requiring a page refresh.

### MIME Type Mapping

- `.png` → `image/png`
- `.jpg`, `.jpeg` → `image/jpeg`
- `.gif` → `image/gif`
- `.webp` → `image/webp`
- `.pdf` → `application/pdf`
- `.html` → `text/html`
- `.json` → `application/json`
- `.csv` → `text/csv`

### Frontend: Verify Assistant Attachment Rendering

The `MessageBubble.tsx` component already renders attachments with image previews, lightbox, and download links. Verify that:

- Attachments on assistant-role messages render the same as on user messages.
- Images show inline thumbnails with lightbox support.
- PDFs, HTML, JSON, CSV show as downloadable file cards.
- If attachments arrive via SSE after the message text is complete, they appear without a full page reload.

### SSE: Deliver Attachments to Frontend

After assistant attachments are created, send an SSE event to the frontend containing the attachment data. The frontend should handle this event and append the attachments to the corresponding message. Consider using the existing `StreamChunk` structure which already has an `attachments` field.

## Edge Cases

- Skip files that don't exist on disk (Claude may reference paths that were deleted or are hypothetical).
- Skip files larger than 10 MB (consistent with user upload limit).
- Don't duplicate — if the same file path appears multiple times in tool events, attach it only once.
- Handle paths with spaces and special characters correctly.
- If the uploads directory doesn't exist, create it.
- Don't attach files from tool results that represent errors or failed operations.
