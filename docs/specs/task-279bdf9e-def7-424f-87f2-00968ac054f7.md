Create a Help page accessible from the navigation menu that documents how Botka works — specifically how chats connect to projects and how context is assembled and loaded into Claude sessions.

## Requirements

### 1. Create Help page (`frontend/src/pages/HelpPage.tsx`)

A new page with well-structured, readable documentation covering these topics:

#### Section: Chat & Projects
- Threads (chats) can optionally be assigned to a project
- When a project is assigned, Claude runs in that project's directory (working directory)
- When no project is assigned, Claude runs in the default directory (`CLAUDE_DEFAULT_WORK_DIR`)
- The project's `CLAUDE.md` content is injected into the context
- Projects are git repositories discovered from the `PROJECTS_DIR` directory
- The sidebar shows a project badge on threads that have a project assigned

#### Section: Context Assembly (this is the key section)
Document the 8-layer hierarchical context system from `internal/claude/context.go`. For each layer, explain:
- What it is
- Where the data comes from (file path or database)
- When it's included (always vs conditional)
- What heading it appears under in the assembled context

The 8 layers in order:
1. **SOUL.md** (Identity) — AI personality/identity from OpenClaw workspace (`$OPENCLAW_WORKSPACE/SOUL.md`). Included if file exists.
2. **USER.md** (About the User) — Information about the user from OpenClaw workspace (`$OPENCLAW_WORKSPACE/USER.md`). Included if file exists.
3. **MEMORY.md** (Operational Memory) — Long-term memory from OpenClaw workspace (`$OPENCLAW_WORKSPACE/MEMORY.md`). Included if file exists.
4. **Daily Notes** (Recent Notes) — Last 3 days of daily memory files from `$OPENCLAW_WORKSPACE/memory/YYYY-MM-DD.md`. Date-sorted, most recent first.
5. **App Memories** (User Preferences) — User-created memories stored in the Botka database (Memories section in Settings). All active memories concatenated.
6. **System Prompt** (Thread Instructions) — From the thread's persona or custom system prompt. Set per-thread.
7. **Project CLAUDE.md** (Project Context) — The CLAUDE.md content from the assigned project. Only included if a project is assigned to the thread.
8. **Conversation History** (Previous Conversation) — Last 200 messages from the thread, each truncated to 500 chars. Only included when starting a new Claude session (not on resume).

Also explain:
- The assembled context is written to a file and passed to Claude via `--append-system-prompt-file`
- Context is only assembled for NEW sessions (when `ClaudeSessionID` is empty). Resumed sessions (via `--resume`) already have their context.
- The session pool pre-warms Claude processes between messages — the next message reuses a warm process, skipping startup time.

#### Section: Session Lifecycle
- New thread → new Claude session → full context assembled
- Subsequent messages → `--resume` with existing session ID → no context reassembly
- Session clear (from UI) → resets session ID → next message creates new session with full context
- Session pool keeps a warm process ready for 5 minutes after each message

#### Section: Task Execution
Brief overview of how tasks differ from chat:
- Tasks run standalone Claude sessions (no `--resume`)
- Tasks run in the project's directory with the project's branch strategy
- Tasks have retry logic and optional verification commands
- Task output is streamed via SSE

#### Section: Keyboard Shortcuts
List available keyboard shortcuts (read from `useKeyboardShortcuts.ts` and `ChatInput.tsx`):
- Ctrl+Shift+O: New chat
- Ctrl+Shift+P: Pin/unpin thread
- Ctrl+Shift+Arrow: Navigate threads
- Ctrl+K: Command palette
- /: Focus input
- Ctrl+L: Focus input
- Shift+Tab: Toggle plan mode (note: currently only works when textarea is focused)
- ?: Show shortcuts modal
- Escape: Close modals/blur input
