## Problem

The `UsageMonitor` (`internal/runner/usage.go`) directly calls the Anthropic OAuth API (`https://api.anthropic.com/api/oauth/usage`) to check rate limits. It handles token refresh from credentials files, which is fragile.

There's a simpler, more reliable option: the `/home/pi/bin/claude-usage` command, which returns the same data as JSON with no auth overhead and no rate limits.

## Requirements

1. **Replace the HTTP API call in `UsageMonitor.Poll()`** with executing `/home/pi/bin/claude-usage` and parsing its JSON output.
2. Keep all existing functionality: thresholds, adaptive polling, `IsRateLimited()`, `CurrentUsage()`, `RefreshUsage()`.
3. Remove the OAuth token refresh logic тАФ no longer needed.
4. Make the command path configurable via an env var (e.g. `CLAUDE_USAGE_CMD` defaulting to `/home/pi/bin/claude-usage`).

## claude-usage output format

```json
{
  "fetched_at": "2026-03-25T13:00:01Z",
  "fetched_epoch": 1774443601,
  "data": {
    "five_hour": {
      "utilization": 35.0,
      "resets_at": "2026-03-25T17:00:00.500386+00:00"
    },
    "seven_day": {
      "utilization": 59.0,
      "resets_at": "2026-03-27T07:00:00.500406+00:00"
    },
    "seven_day_opus": null,
    "seven_day_sonnet": null,
    "extra_usage": {
      "is_enabled": false,
      "monthly_limit": null,
      "used_credits": null,
      "utilization": null
    }
  }
}
```

Note: `utilization` is a percentage (0-100), while the existing `UsageInfo` uses fractions (0.0-1.0). Convert: divide by 100.

## Implementation

### File: `internal/runner/usage.go`

1. Replace the `Poll()` method:
   - Instead of HTTP request to Anthropic API, run `exec.Command(cmdPath)` and capture stdout
   - Parse the JSON output into a struct matching the format above
   - Map `data.five_hour.utilization / 100` тЖТ `FiveHourPct`, same for `seven_day`
   - Map `data.five_hour.resets_at` тЖТ `ResetsAt`
2. Remove all OAuth/token refresh code (credential file reading, token refresh HTTP calls)
3. Add `cmdPath string` field to `UsageMonitor`, set from config

### File: `internal/config/config.go`

1. Add `ClaudeUsageCmd` field (env: `CLAUDE_USAGE_CMD`, default: `/home/pi/bin/claude-usage`)
2. Remove or deprecate `CLAUDE_CREDENTIALS_PATH` if no longer used elsewhere

### File: `cmd/server/main.go`

1. Pass `cfg.ClaudeUsageCmd` to `NewUsageMonitor()` instead of credentials path

### File: `internal/runner/usage_test.go`

1. Update tests тАФ mock the command execution or test the JSON parsing separately

3. Hit `POST /runner/usage/refresh` тАФ should return fresh data
