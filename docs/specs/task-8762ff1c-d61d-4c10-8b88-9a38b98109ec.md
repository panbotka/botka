# MCP Servers Settings Tab

Add a new "MCP Servers" tab to the Settings page where users can manage MCP server configurations (create, edit, delete, toggle default).

## Context

MCP servers are stored in the `mcp_servers` database table with fields: id, name, server_type (stdio/sse), config (JSONB), is_default, active. The API endpoints already exist:
- GET /api/v1/mcp-servers — list all
- POST /api/v1/mcp-servers — create
- PATCH /api/v1/mcp-servers/:id — update
- DELETE /api/v1/mcp-servers/:id — delete

## Requirements

### Settings Tab

Add an "MCP Servers" tab to `frontend/src/pages/SettingsPage.tsx` alongside existing tabs (General, Security, Users, Task Runner, Projects, Personas, Tags, Memories, Voice). Use the `Server` icon from Lucide.

### Server List View

- Display all MCP servers in a list/card layout matching existing settings patterns
- Each server shows: name, type badge ("stdio" / "sse"), default checkbox, active toggle
- Empty state: "No MCP servers configured. Add one to connect external tools to your conversations."
- Add button in the top-right corner

### Add/Edit Form

Show a modal or inline form (match existing patterns like Persona editing) with fields:

**Common fields:**
- Name (text input, required)
- Server Type (radio/select: "stdio" or "sse") — only editable on create, read-only on edit
- Default (checkbox) — "Auto-enable in all conversations and projects"
- Active (toggle) — "Globally enable/disable this server"

**Stdio-specific fields** (shown when type = stdio):
- Command (text input, required — e.g. "npx", "node", "/usr/bin/python3")
- Arguments (text input — comma or space-separated, stored as JSON array)
- Environment Variables (key-value pairs editor — add/remove rows, stored as JSON object)

**SSE-specific fields** (shown when type = sse):
- URL (text input, required — e.g. "https://example.com/mcp/sse")
- Headers (key-value pairs editor — add/remove rows, commonly Authorization header)

### Key-Value Pair Editor

For env vars and headers, create a reusable component:
- List of rows, each with key input + value input + delete button
- "Add" button at bottom to append new row
- Value fields for headers should use password-style input (dots) with a show/hide toggle, since they often contain tokens

### Form Validation

- Name: required, show error if duplicate (409 from API)
- Command (stdio): required when type is stdio
- URL (sse): required when type is sse, must start with http:// or https://

### Delete Confirmation

Show a confirmation dialog before deleting. Warn that this will remove the server from all threads and projects where it's enabled.

### API Client

Add methods to `frontend/src/api/` following existing patterns:
- `listMCPServers()` 
- `createMCPServer(data)`
- `updateMCPServer(id, data)`
- `deleteMCPServer(id)`

## UI Details

- Follow existing SettingsPage patterns for layout, spacing, and component style
- Use Tailwind CSS classes consistent with the rest of the app
- Type badges: use a small colored pill — "stdio" in blue, "sse" in purple
- Default servers should have a visual indicator (star icon or highlighted border)
- Inactive servers should appear dimmed/muted
