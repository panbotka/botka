## Problem

The Projects page (`/projects`) only shows a list of projects with inline settings. There's no way to click through to a project detail view with richer information.

## Requirements

Add a **project detail page** at `/projects/:id` with the following tabs:

### Tab 1: Stats
- Task statistics for this project: total tasks, breakdown by status (pending, queued, running, done, failed)
- Could include charts or just summary numbers
- Average task duration, success rate, etc.

### Tab 2: Git Status
- Show current git status of the project directory — uncommitted changes, untracked files, current branch
- Equivalent of `git status` and `git diff --stat`
- Useful to see if there's work in progress

### Tab 3: Git History
- If the project directory is a git repo, show recent commit history
- Display: commit hash (short), author, date, message
- Backend needs a new endpoint to fetch git log for a project's path

### Tab 4: Tasks
- Show pending, queued, and active (running) tasks for this project
- Reuse task list rendering from the Tasks page
- Clicking a task navigates to its detail page (`/tasks/:id`)

### Tab 5: Settings
- The existing project settings that are currently shown inline on the projects list: branch strategy, verification command, active toggle
- Move or duplicate these here for a more detailed editing experience

## Implementation

### Frontend

**New file:** `frontend/src/pages/ProjectDetailPage.tsx`
- Tabbed layout (Stats | Git Status | Git History | Tasks | Settings)
- Fetch project data, tasks for this project, git status, and git history

**File:** `frontend/src/App.tsx`
- Add route: `/projects/:id` → `ProjectDetailPage`

**File:** `frontend/src/pages/ProjectsPage.tsx`
- Make project names clickable, linking to `/projects/:id`

### Backend

**New endpoint:** `GET /api/v1/projects/:id/git-log`
- Runs `git log --oneline -50` (or similar) in the project's directory
- Returns JSON array of commits: `[{ hash, author, date, message }]`
- Returns empty array or 404 if not a git repo

**New endpoint:** `GET /api/v1/projects/:id/git-status`
- Runs `git status --porcelain` and `git diff --stat` in the project's directory
- Returns JSON: `{ branch, clean (bool), changed_files: [{ path, status }], diff_stat: string }`
- Returns error if not a git repo

**New endpoint (or reuse existing):** `GET /api/v1/projects/:id/stats`
- Returns task counts by status for this project
- Could be a simple SQL query: `SELECT status, count(*) FROM tasks WHERE project_id = ? GROUP BY status`

The tasks endpoint already supports filtering by project, so the Tasks tab can use the existing `GET /api/v1/tasks?project_id=:id` (or however tasks are listed — check existing API).

### Backend handler files

- `internal/handlers/project.go` — add `GetProjectGitLog`, `GetProjectGitStatus`, and `GetProjectStats` handlers
- Register new routes in the appropriate place
