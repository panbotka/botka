Text files and "other" file attachments in chat messages have no download or open link — only PDFs have an "Open" action. Users can't download HTML, text, or other attached files.

## Current state

**`MessageBubble.tsx`** `AttachmentPreviews` component:
- **Images**: clickable thumbnail → lightbox (works)
- **PDFs**: FileCard with "Open" link (`<a href={att.url} target="_blank">Open</a>`) (works)
- **Text files** (text/plain, text/html, text/markdown, etc.): FileCard with NO action — just shows name and size, no way to open or download
- **Other files**: FileCard with NO action — same problem

**Backend**: `GET /api/v1/files/:id/download` endpoint exists and serves files with `Content-Disposition: attachment`. `GET /api/v1/uploads/:filename` serves inline. Both work. The `att.url` field is available on all attachments.

## Requirements

### Add actions to all file types
- **Text files** (line ~146): add "Open" link that opens `att.url` in a new tab (same as PDF), plus a "Download" link
- **Other files** (line ~154): add a "Download" link
- **PDFs**: already have "Open" — optionally add "Download" too for consistency
- **Images**: already work via lightbox — no change needed

### Download link
- Use the download endpoint: `/api/v1/files/${att.id}/download`
- Or use `att.url` with HTML5 `download` attribute: `<a href={att.url} download={att.original_name}>`
- The simpler approach is the `download` attribute — no new API call needed

### HTML files specifically
- HTML files (`text/html`) currently fall into `textAttachments` — "Open" should open them in a new browser tab where they render properly
- This already works via `att.url` + `target="_blank"`
