## Goal

Prevent Botka's `PORT=5110` environment variable from leaking into Claude Code subprocesses. This caused a production crash when a child session ran a dev script that killed the process on `$PORT`.

## Root Cause

Botka sets `PORT=5110` for its HTTP server. Claude Code subprocesses inherit the full environment. When a user ran `make dev` for the memofix project, its `dev.sh` script read `BACKEND_PORT="${PORT:-8080}"`, got `5110`, ran `lsof -ti :5110`, found the Botka PID, and killed it.

## Fix

In both subprocess spawn points, unset `PORT` (and any other Botka-specific env vars that could cause conflicts) from the child process environment:

1. **`internal/claude/runner.go`** — in `Run()`, where `cmd.Env` can be set before `cmd.Start()`
2. **`internal/claude/pool.go`** — in the session spawn logic where the persistent process is started

Set `cmd.Env` to `os.Environ()` filtered to exclude `PORT`. Example:

```go
env := os.Environ()
filtered := make([]string, 0, len(env))
for _, e := range env {
    if !strings.HasPrefix(e, "PORT=") {
        filtered = append(filtered, e)
    }
}
cmd.Env = filtered
```

Also consider excluding other Botka-specific vars that shouldn't leak: `DATABASE_URL`, `MCP_TOKEN`, `SESSION_MAX_AGE`, `WEBAUTHN_ORIGIN`, `WEBAUTHN_RPID`.

## Testing

- `make check` must pass
- Add a unit test verifying that `PORT` is not in the spawned process environment
