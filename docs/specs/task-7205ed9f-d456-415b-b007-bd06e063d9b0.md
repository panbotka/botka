## Problem

The `/clear` command in chat shows "Failed to clear" because the frontend API calls don't match the backend routes.

### Mismatches

| Function | Frontend sends | Backend expects |
|----------|---------------|-----------------|
| `clearSession` | `DELETE /threads/{id}/session` | `POST /threads/{id}/session/clear` |
| `newSession` | `POST /threads/{id}/session` | `POST /threads/{id}/session/new` |

`clearMessages` (`DELETE /threads/{id}/messages`) appears to be correct.

## Fix

Align the frontend API client to match the backend routes.

### File: `frontend/src/api/client.ts`

1. `clearSession` (around line 206): Change from `DELETE /threads/${id}/session` to `POST /threads/${id}/session/clear`
2. `newSession` (around line 210): Change from `POST /threads/${id}/session` to `POST /threads/${id}/session/new`

### Backend routes (for reference, in `internal/handlers/chat.go` ~line 64):
```go
rg.POST("/threads/:id/session/clear", h.ClearSession)
rg.POST("/threads/:id/session/new", h.NewSession)
```
