## Goal

Simplify the usage monitor polling logic. The `claude-usage` command already handles its own caching (cron-refreshed every 10 min), so there's no need for adaptive polling intervals on Botka's side. Just poll every 30 seconds unconditionally.

## Current State

`internal/runner/usage.go` has an `adaptiveInterval()` method (lines 83-96) that adjusts poll frequency based on current utilization:
- < 50% → 60 min
- 50-70% → 30 min
- \> 70% → configurable interval (default 15 min)

This is unnecessary complexity — `claude-usage` reads from a local cache file that's updated by cron every 10 minutes. Calling it more frequently is cheap (just reading a file).

## Fix

1. Remove `adaptiveInterval()` method from `UsageMonitor`
2. In the `Start()` goroutine loop, replace `m.adaptiveInterval()` with a fixed `30 * time.Second` delay
3. The `interval` field and `USAGE_POLL_INTERVAL` env var can be removed (or kept but ignored — removing is cleaner)
4. Update tests in `usage_test.go` that test adaptive intervals — remove them

## Testing

- `make check` must pass
- Verify usage monitor polls at fixed 30s intervals in logs
