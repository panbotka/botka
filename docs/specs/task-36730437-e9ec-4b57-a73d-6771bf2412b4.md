# MCP Server Management — Database Schema & Models

Add database tables and GORM models for managing external MCP (Model Context Protocol) servers that can be attached to chat threads and projects.

## Requirements

### Migration (021_mcp_servers)

Create three tables:

**mcp_servers:**
- `id` BIGSERIAL PRIMARY KEY
- `name` TEXT NOT NULL UNIQUE — display name (e.g. "photo-sorter", "playwright")
- `server_type` TEXT NOT NULL CHECK (server_type IN ('stdio', 'sse')) — transport type
- `config` JSONB NOT NULL DEFAULT '{}' — transport-specific configuration:
  - For stdio: `{"command": "npx", "args": ["@playwright/mcp@latest"], "env": {"KEY": "val"}}`
  - For sse: `{"url": "https://example.com/mcp/sse", "headers": {"Authorization": "Bearer xxx"}}`
- `is_default` BOOLEAN NOT NULL DEFAULT false — if true, auto-enabled in all threads and projects
- `active` BOOLEAN NOT NULL DEFAULT true — global on/off switch
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
- `updated_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()

**thread_mcp_servers** (join table — presence means enabled for that thread):
- `thread_id` BIGINT NOT NULL REFERENCES threads(id) ON DELETE CASCADE
- `mcp_server_id` BIGINT NOT NULL REFERENCES mcp_servers(id) ON DELETE CASCADE
- PRIMARY KEY (thread_id, mcp_server_id)

**project_mcp_servers** (join table — presence means enabled for that project):
- `project_id` UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE
- `mcp_server_id` BIGINT NOT NULL REFERENCES mcp_servers(id) ON DELETE CASCADE
- PRIMARY KEY (project_id, mcp_server_id)

Down migration drops all three tables in reverse order.

### GORM Models

Create `internal/models/mcp_server.go`:

```
type MCPServer struct {
    ID         int64           `json:"id" gorm:"primaryKey"`
    Name       string          `json:"name" gorm:"uniqueIndex;not null"`
    ServerType string          `json:"server_type" gorm:"not null"` // "stdio" or "sse"
    Config     json.RawMessage `json:"config" gorm:"type:jsonb;not null;default:'{}'"`
    IsDefault  bool            `json:"is_default" gorm:"not null;default:false"`
    Active     bool            `json:"active" gorm:"not null;default:true"`
    CreatedAt  time.Time       `json:"created_at"`
    UpdatedAt  time.Time       `json:"updated_at"`
}

type ThreadMCPServer struct {
    ThreadID    int64 `json:"thread_id" gorm:"primaryKey"`
    MCPServerID int64 `json:"mcp_server_id" gorm:"primaryKey"`
}

type ProjectMCPServer struct {
    ProjectID   uuid.UUID `json:"project_id" gorm:"primaryKey;type:uuid"`
    MCPServerID int64     `json:"mcp_server_id" gorm:"primaryKey"`
}
```

Set `TableName()` methods following existing patterns in the codebase (see other model files).

### Validation

- `server_type` must be "stdio" or "sse" (use a const + validation like existing enum patterns in models)
- `name` must be non-empty and unique
- `config` must be valid JSON
