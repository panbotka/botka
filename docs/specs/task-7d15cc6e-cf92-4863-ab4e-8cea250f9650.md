Add bash-style input history to the chat text input. Arrow Up/Down cycles through previously sent messages, just like a terminal command line.

## Behavior (match bash semantics)

- **Arrow Up** (when input is empty or cursor is on the first line): navigate to the previous (older) sent message
- **Arrow Down**: navigate to the next (newer) sent message, or back to empty/draft input
- History includes all sent messages — both regular text and slash commands (e.g. `/clear`)
- When navigating history, the input field content is replaced with the historical message
- If the user has typed something but hasn't sent it yet, pressing Arrow Up saves it as a "draft" — pressing Arrow Down back to the end restores the draft
- Pressing Enter sends the current content (whether from history or typed) and adds it to history
- History resets position to the end after sending

## Detailed Arrow Up/Down rules

- **Arrow Up only activates when the cursor is at the very beginning of the textarea or the textarea has a single line** — if the textarea has multiple lines and the cursor is not on the first line, Arrow Up should do normal cursor movement (move up a line)
- **Arrow Down only activates at the very end of the textarea or single line** — otherwise normal cursor movement
- This ensures multi-line editing still works naturally

## Requirements

### Frontend — Input history state
- Create a `useInputHistory` hook (or add state directly to `ChatInput.tsx`)
- Store history as an array of strings in component state (not persisted — session only is fine)
- Track current history index (position in the array)
- Track draft text (unsent input saved when navigating up)
- Max history size: ~100 entries (drop oldest when exceeded)

### Frontend — Integration into ChatInput
- In `handleKeyDown` in `ChatInput.tsx` (line ~113):
  - Before the existing slash command menu handling
  - If Arrow Up: check cursor position, if at start/single-line → navigate history up
  - If Arrow Down: check cursor position, if at end/single-line → navigate history down
  - `e.preventDefault()` when history navigation activates
- On submit (`handleSubmit`): push the sent content to history, reset index
- The textarea value updates to show the historical message

### Edge cases
- Empty history: Arrow Up does nothing
- At oldest entry: Arrow Up does nothing (stay at oldest)
- At newest entry: Arrow Down restores draft or clears input
- Slash commands are stored in history too (user specifically wants `/clear` → Up → Enter pattern)
- Files are NOT stored in history — only text content
