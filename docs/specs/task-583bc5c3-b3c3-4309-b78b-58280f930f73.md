# Add Per-Thread Background Color to Chat Windows

Users sometimes send messages to the wrong chat thread. Adding a configurable background color to each chat window provides a strong visual cue to prevent this. Each thread can have one of 16 preset background colors, manually changeable from the thread context menu.

## Requirements

### 1. Database: New `color` column on threads

Create migration `019_thread_color.up.sql`:
```sql
ALTER TABLE threads ADD COLUMN color VARCHAR(20) NOT NULL DEFAULT '';
```
And `019_thread_color.down.sql`:
```sql
ALTER TABLE threads DROP COLUMN color;
```

Empty string means "no color" (default/transparent). The value stores a color key like `"blue"`, `"rose"`, `"amber"`, etc. â€” NOT a hex code. This allows the frontend to map each key to different shades for light vs dark themes.

### 2. Backend model

In `internal/models/thread.go`, add to the Thread struct:
```go
Color string `gorm:"size:20;not null;default:''" json:"color"`
```

### 3. Backend handler

The existing `PUT /api/v1/threads/:id` endpoint in `internal/handlers/thread.go` already uses a generic update pattern. Make sure the `color` field is included in the allowed update fields. No new endpoint needed â€” the existing `updateThread` API function on the frontend sends `Partial<Thread>`.

### 4. Frontend type

In `frontend/src/types/index.ts`, add to the `Thread` interface:
```typescript
color?: string
```

### 5. Color palette (16 colors)

Define a color palette constant in a new utility or inline in the component. Each color has a `key`, a `label`, a light-theme background, and a dark-theme background. The dark-theme backgrounds should be very subtle (low opacity or dark shades) so they don't clash with light text.

### 6. ChatView background

In `frontend/src/components/ChatView.tsx`, apply the thread's color to the main chat container (currently at the outer `<div>` around line 765). Use `useSettings()` from `../context/SettingsContext` to get `resolvedTheme` and pick the correct shade.

Apply the background to the scroll container (`ref={scrollContainerRef}` div), not the outermost flex container â€” this keeps the input area and header uncolored while the message area gets the tint.

### 7. Color picker in thread context menu

In `frontend/src/components/ThreadSidebar.tsx`, add a "Color" submenu item in the thread context menu (similar to how "Tags" has a nested submenu). When clicked, it shows the 16 color swatches as small circles in a 4Ă—4 grid plus a "None" option. Clicking a swatch calls `updateThread(threadId, { color: key })` and closes the menu.

Place the "Color" option after "Custom Context" and before "Tags" in the context menu.

### 8. Visual indicator in sidebar

In the thread list sidebar, show a small colored dot or thin left border on threads that have a color set. This gives a visual hint of which color each thread has without opening it.

## Edge Cases

- New threads have no color (empty string) â€” the chat looks exactly as it does today
- The color picker should highlight the currently selected color
- Changing theme should immediately update the background shade (reactive to `resolvedTheme`)
- The background color should be subtle enough that message bubbles, code blocks, and other UI elements remain readable in all themes
