## Požadavek

Botka by měl každou hodinu (v X:05) spustit krátkou Claude Code session s jednoduchým ping promptem. Cíl: udržovat 5h rate limit okno Anthropic API vždy aktivní, aby se nepřerušilo a nemuselo se čekat na nový start.

## Kontext

Anthropic API má 5h rolling window pro rate limity. Pokud se v okně nic nestane, okno "vyprší" a při dalším použití se začíná od nuly. Pravidelný ping zajistí, že okno běží kontinuálně a throughput je konzistentní.

## Implementace

1. **Nový scheduler/cron v runneru** — nezávislý na task queue. Každou hodinu v X:05 spustí jednoduchý Claude Code příkaz.
2. **Ping prompt** — minimální, např. `claude -p "ping"` nebo podobně. Musí to být skutečná API interakce (ne jen lokální příkaz), aby se počítala do 5h okna.
3. **Konfigurace** — ideálně env var pro zapnutí/vypnutí (např. `KEEPALIVE_ENABLED=true`) a interval (`KEEPALIVE_INTERVAL=60m`).
4. **Logging** — logovat že proběhl keepalive ping, ale neplnit logy zbytečnostmi. Stačí jedna řádka za ping.
5. **Neblokovat workery** — ping nesmí zabírat worker slot. Buď dedikovaný goroutine, nebo spouštět mimo executor pool.

## Kde hledat

- Runner loop: `internal/runner/` — přidat keepalive ticker vedle hlavního scheduler loopu
- Claude Code spawn: `internal/runner/executor.go` nebo přímo `os/exec` pro jednoduchý `claude -p "ping"`
- Config: `internal/config/config.go` — přidat keepalive env vars

## Důležité

- Nesmí zabírat worker slot (MAX_WORKERS)
- Nesmí vytvářet task v DB — je to interní mechanismus runneru
- Nesmí běžet pokud je runner stopped
