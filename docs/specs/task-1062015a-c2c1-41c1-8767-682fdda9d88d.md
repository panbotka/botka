Fix the thread sidebar so the archive button and triple-dot (MoreVertical) menu button are always visible on each thread row, instead of only appearing on hover.

## Problem

In `frontend/src/components/ThreadSidebar.tsx`, the action buttons container (around line 327-328) has `opacity-0 group-hover:opacity-100`, making the buttons invisible until hover. This creates an awkward visual gap on the right side of each thread row where the buttons would be.

## Fix

In the `renderThread` function, find this div (around line 327-328):

```tsx
<div
  className="relative flex items-stretch gap-0.5 flex-shrink-0 opacity-0 group-hover:opacity-100 transition-opacity"
  ref={menuOpenId === thread.id ? menuRef : undefined}
>
```

Remove `opacity-0 group-hover:opacity-100 transition-opacity` so the buttons are always visible. The buttons should still have their individual hover color effects (`hover:text-amber-600`, etc.) but should always be rendered and visible.

The result should be:
```tsx
<div
  className="relative flex items-stretch gap-0.5 flex-shrink-0"
  ref={menuOpenId === thread.id ? menuRef : undefined}
>
```

## Verification

- Build the frontend with `cd frontend && npm run build` to ensure no errors
- The archive and triple-dot buttons should be visible on every thread row at all times
- The buttons should still have their hover color effects
- No layout shift when hovering over thread rows
