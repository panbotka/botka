# Add icons to tool calls based on tool name

Show a small icon next to each tool call in the chat streaming UI, derived from the tool name. Use Lucide icons (already in the project).

## Requirements

### Icon mapping
- Create a utility function that maps tool name to a Lucide icon component
- Known tool categories and suggested icons:
  - **File read:** `Read` → FileText, `Glob` → FolderSearch, `Grep` → Search
  - **File write:** `Edit` → Pencil, `Write` → FilePlus
  - **Execute:** `Bash` → Terminal
  - **Agent:** `Agent` → Bot
  - **Todo:** `TodoRead`/`TodoWrite` → ListTodo
  - **Web:** `WebFetch`/`WebSearch` → Globe
  - **MCP tools** (`mcp__*`) → Plug
  - **Notebook:** `NotebookEdit` → BookOpen
  - **Default (unknown tools):** Wrench
- Match by exact name first, then by prefix (e.g. `mcp__` prefix for all MCP tools)
- The mapping should be easy to extend when new tools appear

### Frontend integration
- Show the icon inline next to the tool name in the tool call display during streaming
- Icon should be small (16px or similar), muted color, positioned before the tool name
- Keep existing tool call UI layout — just prepend the icon

## Implementation Notes

- Lucide React icons are already available in the project (`lucide-react`)
- Create the mapping as a simple function in a utils file or co-located with the tool call component
- The icon choices above are suggestions — pick whatever looks best, but ensure every known tool has one and unknown tools get the default Wrench icon
- Existing tests must pass (`make check`)

## Safety

**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.**
