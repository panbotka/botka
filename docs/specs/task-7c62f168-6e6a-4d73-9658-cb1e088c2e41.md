# Preserve Draft Messages Per Thread

## Problem

When the user types a message in a chat thread and switches to another thread, the typed message is lost. The user wants to have draft messages in multiple chats simultaneously.

## Requirements

Store the current input text per thread ID so that switching threads preserves unsent messages.

## Implementation

This is a frontend-only change.

1. **In the chat input component** (likely `ChatInput.tsx` or similar), maintain a `Map<threadId, string>` of draft messages. Use `useRef` or a lightweight state store (not React state that would cause re-renders on every keystroke across threads).

2. **On thread switch:** Before navigating away, save the current input value to the drafts map keyed by the current thread ID. When loading the new thread, restore the draft for that thread (or empty string if none).

3. **On message send:** Clear the draft for that thread from the map.

4. **Storage:** Use in-memory state (React context or a module-level Map) — no need for localStorage persistence. Drafts surviving page reload is not required.

5. **Edge cases:**
   - New thread with no draft → empty input (current behavior)
   - Thread deletion → clean up its draft entry if present
   - Don't interfere with voice input or file attachments
