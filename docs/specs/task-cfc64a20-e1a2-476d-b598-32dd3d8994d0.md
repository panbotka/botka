## Bug

Na homepage (dashboard) se zobrazuje stav workerů jako "stopped", přestože runner aktivně zpracovává tasky (1 running, 11 queued).

## Očekávané chování

Když runner běží a zpracovává tasky, homepage by měla ukazovat odpovídající stav (running/active).

## Kde hledat

1. **Frontend:** Komponenta zobrazující worker status na homepage — pravděpodobně v `frontend/src/pages/` nebo `frontend/src/components/`. Hledej "stopped", "worker", "runner" v UI komponentách.
2. **Backend:** Endpoint který vrací runner status pro homepage — zkontroluj co vrací `/api/v1/runner/status` nebo obdobný endpoint a jak frontend interpretuje odpověď.
3. **SSE:** Stav se může aktualizovat přes SSE — zkontroluj jestli se worker status správně propaguje přes SSE events.

## Reprodukce

1. Otevři homepage
2. Runner běží, zpracovává tasky
3. Homepage ukazuje "stopped" místo skutečného stavu
