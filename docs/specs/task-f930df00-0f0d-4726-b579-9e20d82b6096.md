## Problem

On the Tasks page (`/tasks`), when clicking the action menu (triple-dot / dropdown) on the last task in the list, the dropdown menu gets clipped by the parent container's overflow. The user can't see or interact with the menu options.

## Root Cause

The parent container likely has `overflow: hidden` or `overflow: auto/scroll`, which clips the absolutely-positioned dropdown that extends beyond the container boundaries.

## Requirements

1. The action dropdown menu must be fully visible for ALL tasks, including the last ones in the list.
2. No visual regressions for other tasks.

## Fix Options (pick the simplest that works)

1. **Flip menu direction for bottom items** — detect if the menu would overflow and open it upward instead of downward. Many dropdown libraries support this via a `placement` prop.
2. **Use a portal** — render the dropdown menu in a React portal (`createPortal`) so it's not constrained by parent overflow.
3. **Fix overflow on parent** — change the parent container's `overflow` to `visible` if that doesn't break scrolling. Usually not ideal.
4. **Add bottom padding** — add enough padding at the bottom of the task list container so the dropdown has room. Simplest hack.

## Files to check

- `frontend/src/pages/TasksPage.tsx` — task list rendering, action menu component
- Check for any dropdown/popover component used for task actions
- Check parent container CSS for `overflow` properties

## Verification

1. Run `make check`
2. Open `/tasks`, scroll to the last task, click action menu — it should be fully visible
