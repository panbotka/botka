Set up end-to-end tests using Playwright with the system Chromium browser. Cover critical user flows.

## Important: ARM64 Raspberry Pi
- This runs on ARM64 Raspberry Pi — Playwright's bundled browsers DO NOT WORK
- System Chromium is at `/usr/local/bin/chromium`
- NEVER run `npx playwright install` or try to download browsers
- Configure Playwright to use the system Chromium executable

## Requirements

### Set up Playwright
- Install `@playwright/test` (npm)
- Create `playwright.config.ts`:
  - Use system Chromium: `executablePath: '/usr/local/bin/chromium'`
  - Single browser (chromium only)
  - Base URL: `http://localhost:5110`
  - Timeout: 30s per test
  - Retries: 1
  - Screenshot on failure
- Add `e2e` directory for test files
- Add `make test-e2e` target (requires running app)
- Do NOT add E2E to `make check` — these require a running server, run separately

### Critical user flows to test
1. **Chat flow**: Open chat → create thread → send message → see response streaming → message appears in history
2. **Task flow**: Navigate to tasks → see task list → open task detail → verify task info displayed
3. **Settings flow**: Open settings → change theme → verify theme changes → change font size
4. **Navigation**: All bottom nav tabs work → correct page loads for each
5. **Thread management**: Create thread → rename → delete → verify gone

### Test helpers
- Create a `test-utils.ts` with:
  - `waitForApp()` — waits for the app to be loaded
  - `navigateTo(tab)` — clicks bottom nav tab
  - `createThread(name)` — creates a new chat thread
  - Common selectors as constants

### Test data
- Tests should be independent — each test creates its own data
- Clean up after tests (delete created threads/tasks)
- Don't depend on existing data in the database

## Implementation Notes
- Playwright with `executablePath` works fine without installing browsers
- Tests run against the actual running app on port 5110
- For CI, the app needs to be running first — document this in the Makefile
- Use `page.waitForSelector` and `page.waitForResponse` for async operations
- Keep tests focused — one flow per test file
- Existing `make check` must still pass (E2E is separate)

## Safety
**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.**
**NEVER run `npx playwright install` — it will fail on ARM64.**
