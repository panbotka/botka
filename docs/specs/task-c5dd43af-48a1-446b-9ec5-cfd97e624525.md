# Fix Thread Sources: Add Button Does Nothing

## Problem

Clicking "Add source" in the ThreadSourcesEditor does nothing. The root cause:

1. Frontend (`frontend/src/components/ThreadSourcesEditor.tsx` line 34-37) calls `createThreadSource(threadId, { url: '', label: '' })` — sends empty URL
2. Backend (`internal/handlers/thread_source.go` line 64) rejects the request: `if err := c.ShouldBindJSON(&req); err != nil || req.URL == ""` returns 400 "url is required"
3. Frontend `handleAdd` doesn't catch the error, so the failure is silent — no new row appears

## Requirements

Fix this so the "Add source" button works. Use the **optimistic local row** approach:

### Frontend Changes (`frontend/src/components/ThreadSourcesEditor.tsx`)

- When the user clicks "Add source", add a temporary local-only row to the `sources` state with a temporary negative ID (e.g. `Date.now() * -1`) and empty url/label
- Do NOT call the backend `createThreadSource` immediately
- The new empty row appears in the list with the url and label inputs ready to type into
- When the user fills in the URL and blurs (triggers `handleSave`):
  - If the source has a temporary (negative) ID AND the URL is non-empty: call `createThreadSource` to persist it, then replace the temp ID with the real ID from the response
  - If the source has a temporary (negative) ID AND the URL is still empty: do nothing (keep the local row, or remove it on blur — either is fine)
  - If the source has a real (positive) ID: call `updateThreadSource` as before
- When deleting a temporary (not yet persisted) source, just remove it from local state without calling the backend
- Auto-focus the URL input when a new source row is added

### Error Handling

- If `createThreadSource` fails, show an inline error or remove the temp row and show a brief error message
- If `updateThreadSource` fails, revert the local change or show an error

### No Backend Changes Needed

The backend validation is correct — URL should be required for persisted sources. The fix is entirely in the frontend.
