# Render URLs in User Messages as Clickable Links

URLs typed in user chat messages are displayed as plain text and are not clickable. They should be rendered as clickable `<a href>` links that open in a new tab.

## Root Cause

In `frontend/src/components/MessageBubble.tsx` (around line 330), user messages are rendered as plain text inside a `<p>` tag:

```tsx
<p className="whitespace-pre-wrap text-[15px] leading-relaxed">{message.content}</p>
```

Assistant messages go through `<MarkdownContent />` which uses ReactMarkdown and renders links correctly. User messages bypass this entirely.

## Requirements

- URLs in user messages should be automatically detected and rendered as clickable `<a>` tags
- Links should open in a new tab (`target="_blank" rel="noopener noreferrer"`)
- The detection should handle common URL patterns: `https://...`, `http://...`, and bare `www.` prefixes
- The rest of the user message text should remain plain text (do NOT pass user messages through full ReactMarkdown — keep them visually as plain text, just with clickable links)
- Links should be styled subtly — underlined with a slightly different color (e.g., `text-blue-600 underline` in light mode, `text-blue-400 underline` in dark mode), matching how links typically look in chat apps
- Preserve the existing `whitespace-pre-wrap` behavior so line breaks and spacing still work

## Implementation Approach

Create a small utility function (e.g., `linkifyText`) that takes a string and returns React elements — splitting the text on URL patterns and wrapping matches in `<a>` tags while keeping non-URL parts as plain text spans. Use this in the user message rendering path instead of rendering `{message.content}` directly.

A simple regex like `/https?:\/\/[^\s<>]+|www\.[^\s<>]+/g` should work for detection. Strip trailing punctuation (`.`, `,`, `)`, `]`) that's likely not part of the URL.

## Files to Change

- `frontend/src/components/MessageBubble.tsx` — replace plain text rendering of user messages with linkified version
- Optionally create a small utility in `frontend/src/utils/` for the linkify function if it would be reused

## Edge Cases

- Multiple URLs in one message should all be linkified
- URLs at the start, middle, or end of text should work
- URLs followed by punctuation (period, comma) should not include the trailing punctuation in the link
- Messages with no URLs should render identically to today
- Very long URLs should not break the layout (CSS `break-all` or `overflow-wrap: anywhere` on the link)
