# MCP Server Toggle UI for Threads and Projects

Add UI controls for enabling/disabling MCP servers per thread (conversation) and per project.

## Context

MCP servers are managed in Settings. Each server has an `is_default` flag — default servers are auto-enabled everywhere. Non-default servers need manual enabling per thread or project. The API endpoints already exist:
- GET /api/v1/threads/:id/mcp-servers — list servers with `enabled` boolean
- PUT /api/v1/threads/:id/mcp-servers — set enabled server IDs `{"mcp_server_ids": [1,3]}`
- GET /api/v1/projects/:id/mcp-servers — same for projects
- PUT /api/v1/projects/:id/mcp-servers — same for projects

Each response item: `{"id", "name", "server_type", "is_default", "active", "enabled"}`. The `enabled` field is true if the server is default OR explicitly assigned.

## Requirements

### Thread MCP Toggle

Add an MCP servers section to the thread settings area. Look at how thread settings are currently displayed (thread sidebar, settings panel, or modal — check the existing thread editing UI in the codebase and add to the same location).

**Display:**
- Section header: "MCP Servers" with a plug/server icon
- List all active MCP servers as toggle switches
- Default servers: toggle is ON and disabled (cannot be turned off) with a label "(default)"
- Non-default servers: toggle is interactive, ON/OFF
- Each row shows: server name, type badge (stdio/sse), toggle
- If no MCP servers exist at all, show: "No MCP servers configured" with a link to Settings

**Behavior:**
- On toggle change, immediately call PUT /api/v1/threads/:id/mcp-servers with updated list of enabled non-default server IDs
- Show a brief toast/notification on success
- Toggling MCP servers should indicate that the change takes effect on the next message (existing session will be refreshed)

### Project MCP Toggle

Add an MCP servers section to the project detail/edit page. Look at `ProjectDetailPage.tsx` or wherever project settings are edited.

**Display:**
- Same pattern as thread: list of servers with toggles
- Default servers locked ON with "(default)" label
- Non-default servers toggleable

**Behavior:**
- On toggle change, call PUT /api/v1/projects/:id/mcp-servers
- Changes affect all new chat sessions and task executions for this project

### API Client

Add methods to `frontend/src/api/` following existing patterns:
- `getThreadMCPServers(threadId)`
- `setThreadMCPServers(threadId, serverIds)`
- `getProjectMCPServers(projectId)`
- `setProjectMCPServers(projectId, serverIds)`

### Custom Hook

Create `frontend/src/hooks/useMCPServers.ts`:
- Fetch MCP servers for a given thread or project
- Provide a toggle function that optimistically updates UI and calls the API
- Handle loading and error states

## Edge Cases

- Thread with no project: only show default + thread-specific servers
- Thread with project: show default + project-enabled + thread-specific (all merged)
- Server deleted while viewing: handle gracefully (refetch on 404)
- No MCP servers configured globally: show empty state with link to settings
- All servers are default: show list but all toggles disabled (informational only)

## UI Details

- Match existing component patterns in the codebase (check similar toggle lists like tag assignment or persona selection)
- Use Tailwind CSS consistent with the rest of the app
- Type badges: small pill — "stdio" blue, "sse" purple (same as in Settings tab)
- Keep the toggle section collapsed by default if there are no non-default servers to configure
