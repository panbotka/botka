## Task 6: MCP Server (Tasks Only)

Port Saiduler's MCP (Model Context Protocol) server to Botka.

### Context
The MCP server allows Claude Code sessions to create and manage tasks in Botka via tool calls. It supports both SSE transport (via HTTP) and stdio transport (for direct CLI integration). Only task-related tools are exposed — no chat tools.

### Reference Files
- `/home/pi/projects/saiduler/internal/mcp/server.go` — MCP server core, tool definitions
- `/home/pi/projects/saiduler/internal/mcp/tools.go` — Tool implementations (create_task, list_tasks, get_task, update_task, list_projects, get_runner_status, start_runner)
- `/home/pi/projects/saiduler/internal/mcp/sse.go` — SSE transport handler for Gin

### What to Create

1. **`internal/mcp/server.go`** — Port from Saiduler. MCP server with JSON-RPC 2.0 protocol. Tool definitions for task management.

2. **`internal/mcp/tools.go`** — Port all tool implementations:
   - `create_task` — Create a task with title, project_name, spec, status, priority
   - `list_tasks` — List/filter tasks by project or status
   - `get_task` — Get task details by UUID
   - `update_task` — Update status, spec, priority, or title
   - `list_projects` — List available projects
   - `get_runner_status` — Get runner state
   - `start_runner` — Start the runner

3. **`internal/mcp/sse.go`** — Port SSE transport handler for Gin:
   - `GET /mcp/sse` — Establish SSE connection
   - `POST /mcp/message` — Send JSON-RPC messages

4. **Wire in main.go:**
   - Register `/mcp` routes in setupRouter
   - Implement `runMCP()` for stdio mode (the "mcp" subcommand)

5. Update import paths from saiduler to botka.

### Verification
