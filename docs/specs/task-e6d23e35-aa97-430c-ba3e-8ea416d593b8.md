## Goal

Allow users to hide individual messages in a chat thread. Hidden messages are NOT deleted — they remain in the database but are visually hidden in the UI. The user should be able to unhide them as well.

## Requirements

1. **Database**: Add a `hidden` boolean column (default `false`) to the `messages` table. Create a migration.
2. **Backend API**: Add endpoint `PATCH /api/v1/messages/:id/hide` that toggles the `hidden` field. Return the updated message.
3. **Frontend — message action**: Add a "Hide" option to the message context menu / action buttons (the same area where edit, regenerate, branch actions are). For hidden messages, show "Unhide" instead.
4. **Frontend — display**: Hidden messages should be collapsed to a single small line like "Message hidden" (muted, subtle styling) with a click/button to unhide or expand. Do NOT remove them from the DOM entirely — the user needs to know something is there and be able to restore it.
5. **API response**: The `hidden` field should be included in message JSON responses. The frontend filters the display, not the backend query — all messages (including hidden) are always returned.

## Implementation Hints

- Message model is in `internal/models/message.go`
- Message handlers are in `internal/handlers/message.go`
- Message rendering is in `frontend/src/components/MessageBubble.tsx`
- Message actions (edit, regenerate, branch) are wired in `frontend/src/components/ChatView.tsx`
- Create migration with `make migrate-create NAME=add_message_hidden`
