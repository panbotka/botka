## Problem

Clicking "Archive" on a thread archives it immediately without any confirmation. Same for "Delete" — no confirmation either. Both actions should require confirmation.

## Requirements

1. **Move Archive into the triple-dot (⋯) menu** on the thread item, right above "Delete"
2. **Add confirmation dialog for Archive** — e.g. "Are you sure you want to archive this thread?"
3. **Add confirmation dialog for Delete** — e.g. "Are you sure you want to delete this thread?"
4. If confirmed, proceed with the action. If cancelled, do nothing.

## Implementation

Find the thread item rendering in the sidebar (likely `ThreadSidebar.tsx`). The triple-dot menu already exists with a "Delete" option. Add "Archive" as a new menu item right above "Delete". Both actions should use `window.confirm()` before executing.
