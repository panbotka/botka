## Problem

When a chat thread has an assigned project (e.g. "botka"), the AI assistant doesn't know about it. When the user asks to create tasks, the AI asks "for which project?" — which is redundant because the thread already has a project assigned.

## Solution

Include the thread's assigned project name in the context that gets assembled and sent to Claude Code sessions. The context assembly is in `internal/claude/context.go`.

When a thread has an assigned project, add a clear note in the context like:

```
This chat is associated with project "botka" (/home/pi/projects/botka).
When creating tasks or discussing code, default to this project unless the user specifies otherwise.
```

### Implementation Steps

1. Look at `internal/claude/context.go` to understand how context is assembled
2. Look at `internal/handlers/chat.go` to see how the thread's project info is passed to the context builder
3. The thread model has a `ProjectID` field — resolve it to the project name and path
4. Add the project info as a section in the assembled context (after the project's CLAUDE.md, or as a standalone note)
5. Make sure the note is clear enough that the AI will use it as the default project for task creation, code discussion, etc.

## Files to Modify

- `internal/claude/context.go` — add project context section
- Possibly `internal/handlers/chat.go` — ensure project data is available during context assembly

## Testing

- Assign a project to a chat thread
- Start a new message asking to create tasks
- Verify the AI defaults to the assigned project without asking
- Run `make check`

## Safety

**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.** Task agents run inside Botka — deploying or restarting would kill the agent's own process. Only build and test.
