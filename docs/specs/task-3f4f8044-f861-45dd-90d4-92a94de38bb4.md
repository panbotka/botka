# Fix layout shift from streaming indicator dot in thread sidebar

The orange pulsing dot that indicates AI is generating a response causes layout shift in the thread sidebar. It's rendered in a separate row (`div` at line 283) that adds/removes height when streaming starts/stops, making the entire sidebar jump.

## Requirements

- Move the streaming indicator dot to be **inline before the thread title** (inside the title `span` at line 269, alongside the pin icon and persona icon)
- The thread sidebar item height must remain constant whether the dot is visible or not
- The dot should only show when the thread is streaming AND is not the active thread (preserve existing condition at line 284)
- Keep the existing dot styling: `w-1.5 h-1.5 rounded-full bg-amber-500 animate-pulse flex-shrink-0`
- Do NOT change the tags row or any other sidebar layout

## Implementation Notes

- The fix is in `frontend/src/components/ThreadSidebar.tsx` around lines 269-286
- Move the streaming dot into the existing `flex items-center gap-1` span that contains the pin icon and persona icon (line 269)
- Remove the separate `div` wrapper (lines 283-286) that currently holds the dot
