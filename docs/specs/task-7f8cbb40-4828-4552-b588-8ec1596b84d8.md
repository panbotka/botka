## Goal

The Tasks page feels stale because it doesn't re-fetch data often enough. Add three refresh triggers:

1. **Tab switch**: Fetch tasks every time the user navigates to the Tasks tab/page
2. **Polling**: Fetch tasks every 10 seconds while the Tasks page is visible
3. **Window focus**: Fetch tasks when the browser tab regains focus (e.g., user switches back from another app/tab)

## Current State

The Tasks page is at `frontend/src/pages/TasksPage.tsx`. Check how it currently fetches tasks — likely a single `useEffect` on mount. The chat page already has a `useRefreshOnFocus` hook (see `frontend/src/hooks/useRefreshOnFocus.ts`) that can be reused or referenced.

## Implementation

1. **On mount / tab switch**: The page component should fetch tasks in a `useEffect` that runs when the component mounts (which happens on every navigation to the page). This likely already works — verify.

2. **Polling every 10s**: Add a `setInterval` that calls the fetch function every 10 seconds. Clear it on unmount. Only poll while the page is visible (use `document.visibilityState`).

3. **Window focus**: Use the existing `useRefreshOnFocus` hook or add a `focus`/`visibilitychange` event listener that triggers a re-fetch when the page becomes visible.

Make sure to avoid race conditions — if a fetch is already in flight, don't start another one. Use an abort controller or a simple flag.
