## Problem

The Tasks page (`/tasks`) only filters by status (tabs). The user wants to also filter by project, so they can focus on tasks for one project — e.g. to reprioritize and queue them.

## Requirements

1. **Add a project filter dropdown** on the Tasks page, next to or below the status tabs.
2. Options: "All projects" (default) + list of all projects that have tasks.
3. Filter is client-side — tasks are already fetched, just filter in memory.
4. **Combine with status tabs** — both filters apply together (e.g. "pending" + "botka" shows only pending botka tasks).
5. **Persist in URL** — add `?project=<name>` query param alongside the existing `?status=` param (from the deep links task). If the deep links task hasn't been done yet, just use `useSearchParams` for both.
6. Task counts on status tabs should reflect the project filter (i.e. if filtering by "botka", tab counts show only botka task counts).

## Implementation

### File: `frontend/src/pages/TasksPage.tsx`

1. Add state for selected project (or read from `useSearchParams`)
2. Fetch the project list from API or derive unique projects from the tasks data
3. Add a `<select>` or dropdown component for project selection, placed near the status tabs
4. Filter the tasks list by both `activeFilter` (status) and selected project before rendering
5. Update tab counts to reflect both filters

### API

The tasks already include `project_id` and project info. No backend changes needed — this is purely frontend filtering.

## Verification

1. Run `make check`
2. Select a project — only tasks for that project should show
3. Combine with a status tab — both filters apply
4. "All projects" shows everything as before
