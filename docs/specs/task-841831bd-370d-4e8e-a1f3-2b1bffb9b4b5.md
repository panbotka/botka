# Improve Chat Command Buttons UX

The `CommandButtons` component (`frontend/src/components/CommandButtons.tsx`) and `useProjectCommands` hook (`frontend/src/hooks/useProjectCommands.ts`) already provide basic Start/Stop functionality for dev and deploy commands. Improve the UX to make it a proper mini process manager for dev servers.

## Current State

- `CommandButtons.tsx` renders Play/Stop buttons for dev and deploy
- When running, shows a green pulsing dot + Stop icon
- PID and port info is only in the title tooltip, not visible
- No restart button — user has to stop, wait, then start again
- Toast shows briefly after start/stop

## Requirements

### Running State Display

When a dev server is running, the button group should clearly show:
- The green pulsing dot (keep this — it's good)
- The address as a clickable link: `pi:8093` (hostname `pi`, port from the running command status). The link should open `http://pi:8093` in a new tab.
- The PID in smaller text or tooltip: "(PID 12345)"

The hostname is always `pi` — hardcode it or derive it from `window.location.hostname` (the Botka UI runs on the same host).

### Button Actions for Running Dev

When dev is running, show TWO buttons instead of one:
1. **Restart** button (RefreshCw icon) — kills the current process and immediately starts a new one. This is the most common action (user made changes, wants to restart the dev server). Show a brief "Restarting..." state.
2. **Stop** button (Square icon) — stops the process without restarting

Layout when running: `[pulsing dot] [pi:8093] [↻ Restart] [■ Stop]`

Where `pi:8093` is a clickable link to `http://pi:8093`.

Layout when stopped: `[▶ Dev]` (single button, same as now)

### Button Actions for Deploy

Deploy is simpler — it's a one-shot action:
- When not running: `[🚀 Deploy]` button (same as now, with confirmation)
- When running: `[pulsing dot] Deploying... [■ Stop]`
- No restart needed for deploy

### Restart Logic

Add a `handleRestart` function to the `useProjectCommands` hook:
1. Kill the current process (`killProjectCommand`)
2. Wait briefly (1-2 seconds) for the port to free up
3. Start a new instance (`runProjectCommand`)
4. Show toast: "Dev restarting..."
5. Poll immediately after to update status

### Polish

- Buttons should be compact enough to fit in the chat header without taking too much space
- On mobile (small screens), show only icons without labels (the `hidden sm:inline` pattern is already there, keep it)
- The toast should not overlap with the buttons — keep it inline as it is now
- If no port is configured, show just "Dev running (PID 12345)" without a link

## Files to Modify

- `frontend/src/components/CommandButtons.tsx` — UI changes, add restart button, add clickable address link
- `frontend/src/hooks/useProjectCommands.ts` — add `handleRestart` function
- No backend changes needed — all existing API endpoints support this
