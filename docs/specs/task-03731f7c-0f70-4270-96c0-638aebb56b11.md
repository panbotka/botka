## Task 12: Chat Frontend — ChatPage and Thread Sidebar

Create the ChatPage that integrates the thread list sidebar with the ChatView, and wire up all chat API streaming.

### Context
The Chat section is a split-panel layout: thread list on the left, chat view on the right. On mobile, it collapses to single-panel navigation. This task wires together the components from Task 11 with the API layer.

### Reference Files
- `/home/pi/projects/chatovadlo/frontend/src/components/ChatLayout.tsx` — Main responsive layout
- `/home/pi/projects/chatovadlo/frontend/src/components/Sidebar.tsx` — Thread list with search, filters, folders
- `/home/pi/projects/chatovadlo/frontend/src/api/` — API client with SSE streaming functions
- `/home/pi/projects/chatovadlo/frontend/src/components/BottomNav.tsx` — Mobile bottom navigation

### What to Create

1. **`pages/ChatPage.tsx`** — The main chat page:
   - Split layout: thread sidebar (left, ~300px) + ChatView (right, flex)
   - On mobile (useIsMobile): single panel, toggle between thread list and chat
   - New chat button in sidebar header
   - Thread selection navigates to `/chat/:id`
   - Empty state when no thread selected

2. **Thread Sidebar** (component within ChatPage or separate):
   - Thread list sorted by threadOrder utility (pinned first, then by updated_at)
   - Search input that filters threads by title
   - Filter by: tag, project, archived
   - Each thread item shows: title, last message preview (truncated), time ago, pin indicator, tag colors
   - Right-click context menu or swipe actions: rename, pin/unpin, archive, delete
   - "New Chat" button at top
   - Persona quick-start: show persona avatars, click to create thread with that persona

3. **Chat API streaming** in `api/client.ts`:
   - `streamChat(threadId, content, files?, planMode?)` — POST to `/api/v1/threads/:id/messages`, parse SSE events
   - `streamRegenerate(threadId)` — POST to `/api/v1/threads/:id/regenerate`
   - `streamEdit(threadId, messageId, content)` — POST to `/api/v1/threads/:id/messages/:mid/edit`
   - `streamBranch(threadId, parentMessageId, content)` — POST to `/api/v1/threads/:id/branch`
   - SSE event parsing: `data:` for content, `event: thinking` for thinking, `event: tool_use`, `event: tool_result`, `event: usage`, `event: error`, `event: title`
   - All streaming functions should return an EventSource-like interface or use fetch with ReadableStream

4. **Wire routes in App.tsx:**
   - `/chat` → ChatPage (no thread selected)
   - `/chat/:id` → ChatPage (thread selected, load messages)

5. **Thread CRUD API methods** in client.ts:
   - createThread, getThread, listThreads, renameThread, deleteThread
   - pinThread, unpinThread, archiveThread, unarchiveThread
   - setThreadModel, setThreadProject, setThreadTags
   - clearMessages, clearSession, newSession

6. **Mobile bottom navigation** — BottomNav component:
   - Show on mobile only
   - Icons for: threads list, new chat, settings
   - Replaces sidebar on small screens

### Verification
- Navigate to /chat → see thread sidebar with search
- Click "New Chat" → creates thread, opens chat view
- Type message → streams response in real-time with content deltas
- Thinking sections appear and are collapsible
- Tool calls show inline panels
- Edit message → creates branch, shows branch indicator
- Switch branches → different conversation path
- Search threads by title works
- Filter by tag/project works
- Mobile: single-panel layout with bottom nav
- Pin/archive/delete from context menu works
