## Task 15: Docker, Systemd, and Deployment

Create deployment configuration for Botka.

### Context
Botka needs to be deployable via Docker and/or systemd service on the Raspberry Pi. The PostgreSQL database runs on the shared instance at localhost:5432.

### Reference Files
- `/home/pi/projects/saiduler/Dockerfile` — Multi-stage Docker build
- `/home/pi/projects/saiduler/docker-compose.yml` — Docker Compose config
- `/home/pi/projects/saiduler/packaging/saiduler.service` — Systemd unit
- `/home/pi/projects/chatovadlo/Dockerfile` — Chatovadlo's Docker build
- `/home/pi/projects/chatovadlo/docker-compose.yml` — Chatovadlo's compose

### What to Create

1. **`Dockerfile`** — Multi-stage build:
   - Stage 1 (node:22-alpine): Build frontend (`npm ci && npm run build`)
   - Stage 2 (golang:1.25-alpine): Build Go binary with embedded frontend
   - Stage 3 (alpine:3.21): Runtime with minimal image
   - Copy binary as `/server`
   - Expose port 5110
   - CMD ["/server"]

2. **`docker-compose.yml`**:
   ```yaml
   services:
     botka:
       build: .
       container_name: botka
       ports:
         - "5110:5110"
       env_file: .env
       volumes:
         - /home/pi/projects:/home/pi/projects:rw
         - /home/pi/.claude:/home/pi/.claude:ro
         - /home/pi/.openclaw:/home/pi/.openclaw:ro
         - uploads:/app/data/uploads
         - context:/app/data/context
       networks:
         - shared-db
       restart: unless-stopped

   volumes:
     uploads:
     context:

   networks:
     shared-db:
       external: true
   ```

3. **`packaging/botka.service`** — Systemd unit:
   ```ini
   [Unit]
   Description=Botka - AI Task Scheduler & Chat
   After=network.target postgresql.service

   [Service]
   Type=simple
   User=pi
   WorkingDirectory=/home/pi/projects/botka
   EnvironmentFile=/home/pi/projects/botka/.env
   ExecStart=/usr/local/bin/botka
   Restart=on-failure
   RestartSec=5

   [Install]
   WantedBy=multi-user.target
   ```

4. **Create the `botka` database** on shared PostgreSQL:
   ```sql
   CREATE USER botka WITH PASSWORD 'botka';
   CREATE DATABASE botka OWNER botka;
   ```
   Add this as a note in the CLAUDE.md or README, and create a `scripts/create-db.sql` file.

5. **Update Makefile** deploy targets:
   - `make deploy` — build binary, stop systemd service, copy to /usr/local/bin/botka, start service
   - `make install-service` — copy service file, enable service
   - `make docker-build` — docker compose build
   - `make docker-up` — docker compose up -d

6. **Create `.env`** file (from .env.example) with actual values for the Pi deployment.

### Verification
- `docker compose build` succeeds
- `docker compose up` starts Botka on port 5110, connects to shared PostgreSQL
- Systemd service starts and runs correctly
- App accessible at http://localhost:5110
- Database tables created on first startup
