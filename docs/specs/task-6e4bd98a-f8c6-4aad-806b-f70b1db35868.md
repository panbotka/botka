# Frontend UI for Signal Bridge Management

Add UI elements to the thread settings/sidebar for managing Signal group bridge connections.

## Requirements

### Thread Signal bridge panel

In the thread settings area (where project, persona, tags, and sources are managed), add a "Signal Bridge" section:

- Show current bridge status: connected group name or "Not connected"
- A dropdown/selector to pick from available Signal groups (fetched from `GET /api/v1/signal/groups`)
- "Connect" button to link the selected group (`PUT /api/v1/threads/:id/signal-bridge`)
- "Disconnect" button when already connected (`DELETE /api/v1/threads/:id/signal-bridge`)
- Visual indicator (small icon or badge) showing the bridge is active

### Visual indicator on thread list

- Threads with an active Signal bridge should show a small Signal icon or indicator in the thread list sidebar, so it's immediately visible which threads are bridged

### Visual indicator on messages

- Messages that originated from Signal (user messages with the `**Name:** text` format from the bridge) could be visually distinguished — but keep it subtle. The sender name prefix in the message content is sufficient for MVP.

### API integration

Add API client methods in `frontend/src/api/`:

- `getSignalGroups()` → `GET /api/v1/signal/groups`
- `getSignalBridge(threadId)` → `GET /api/v1/threads/:id/signal-bridge`
- `setSignalBridge(threadId, groupId, groupName)` → `PUT /api/v1/threads/:id/signal-bridge`
- `removeSignalBridge(threadId)` → `DELETE /api/v1/threads/:id/signal-bridge`

### Error states

- Signal-cli unavailable: show "Signal service unavailable" instead of group dropdown
- No groups found: show "No Signal groups found"
- Connection failure: show error toast

### Design

Follow the existing design patterns in the codebase — Tailwind CSS, Lucide icons, same component structure as other thread settings panels. Use a subtle Signal-like icon (MessageSquare from Lucide or similar) for the bridge indicator.
