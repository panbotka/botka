# Make task creation response clickable with link to task detail

When a chat message contains a task creation confirmation (from the `create_task` MCP tool), render the task ID as a clickable link that opens the task detail page in a new tab.

## Requirements

### Detection
- Detect task creation results in assistant messages — the MCP tool result from `mcp__botka__create_task` contains text like `Created task <uuid> 'Title' for project ...`
- Parse the task UUID from this text

### Rendering
- Render the task UUID (or the whole confirmation line) with a clickable link to `/tasks/<uuid>`
- Link opens in a new tab (`target="_blank"`)
- Style the link subtly — underline or colored text, consistent with the app's design language

### Approach
- This should work in the rendered markdown/message content, not just during streaming
- The simplest approach: post-process assistant message content and convert task UUID patterns into links
- Pattern to match: `Created task <uuid>` where UUID is a standard format (8-4-4-4-12 hex)
- Alternatively, detect the tool result event for `create_task` and enrich it with a link

## Implementation Notes

- The task detail page route is `/tasks/:id` (already exists)
- Keep it simple — a regex replacement in the message renderer is sufficient
- Make sure it works both in streaming (live) and in rendered history (persisted messages)
- Existing tests must pass (`make check`)

## Safety

**NEVER run `make deploy`, `make install-service`, `systemctl restart botka`, or any command that would restart the Botka service.**
