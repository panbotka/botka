# Fix Hide Message Causing Entire Page to Disappear

When a user clicks "hide message" in the chat UI, the entire page content disappears instead of just collapsing that one message into a small "Message hidden" button. After a browser refresh, the page renders correctly — the hidden message shows as a small "Message hidden" button and all other messages are visible. This means the backend toggle works, but the frontend state update or re-render is broken.

## Symptom

1. User clicks the hide (eye-off) button on any message
2. The entire chat view goes blank — all messages disappear
3. After browser refresh (F5), the page loads correctly: the hidden message shows the small "Message hidden" pill, all other messages are visible

## Investigation Areas

### 1. The handleHide function (`frontend/src/components/ChatView.tsx`, line 620-627)

```
const handleHide = useCallback(async (messageId: number) => {
  try {
    const updated = await toggleMessageHidden(messageId);
    setMessages((prev) => prev.map((m) => m.id === messageId ? { ...m, hidden: updated.hidden } : m));
  } catch {
    // ignore
  }
}, []);
```

The code looks correct — it maps over messages, only updating the target message's `hidden` field. But investigate:
- Does `toggleMessageHidden` return the expected shape? Check if `updated.hidden` is actually a boolean.
- Could `updated` be undefined/null, causing `updated.hidden` to throw? The try-catch should catch this, but the error is silently ignored.
- Could `setMessages` be called with a corrupted array?

### 2. ErrorBoundary is unused (`frontend/src/components/ErrorBoundary.tsx`)

The `ErrorBoundary` component exists but is **never imported or used anywhere** in the app. If ANY error occurs during the re-render triggered by `setMessages`, the entire React component tree will crash and show a blank page. This is a critical gap.

Check if the re-render after hiding a message throws an error — for example, if `MessageBubble` tries to access a property that doesn't exist on the updated message object.

### 3. Race condition with BroadcastChannel (`frontend/src/hooks/useChatSync.ts`)

The `handleSyncThreadUpdated` callback (ChatView.tsx line 88-96) refetches ALL messages and calls `setMessages(data.messages)`. If another tab triggers a `thread-updated` broadcast while the hide is in progress, it could overwrite the messages state.

### 4. MessageBubble rendering for hidden messages (`frontend/src/components/MessageBubble.tsx`, line 231-244)

When `message.hidden` is true, MessageBubble returns an early render with a small button. Verify this code path doesn't throw.

## Required Fix

1. **Find and fix the root cause** — reproduce the bug by hiding a message and observe what happens in the browser console. Look for React errors, unhandled exceptions, or state corruption.

2. **Wrap the app with ErrorBoundary** — even after fixing this specific bug, add the existing `ErrorBoundary` component around the main app content in `frontend/src/App.tsx` to prevent future render errors from blanking the entire page.
