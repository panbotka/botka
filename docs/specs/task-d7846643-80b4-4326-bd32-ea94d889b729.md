# Signal Client Go Package

Go package that wraps the signal-cli JSON-RPC over HTTP API, providing a clean interface for sending and receiving Signal messages.

## Requirements

- Create package `internal/signal/` with a `Client` struct
- Client connects to signal-cli daemon's HTTP JSON-RPC endpoint (default `http://127.0.0.1:5107/api/v1/rpc`)
- Configuration via new env var `SIGNAL_CLI_URL` (add to `internal/config/config.go`), default `http://127.0.0.1:5107`
- All methods use JSON-RPC 2.0 protocol: `POST` with `{"jsonrpc":"2.0","method":"...","params":{...},"id":N}`

### Methods to implement

1. **ListGroups()** — returns list of Signal groups with their IDs, names, and member counts
   - JSON-RPC method: `listGroups`
   - No params required
   - Group ID is base64-encoded string (e.g. `4vnBAIsEAQqiNuELxVNJqfhDzHz4ZwMn9T0BjZtr+ng=`)

2. **SendGroupMessage(groupID string, message string)** — sends a text message to a Signal group
   - JSON-RPC method: `send`
   - Params: `{"groupId": "<base64-group-id>", "message": "<text>"}`
   - Returns timestamp and per-recipient delivery status

3. **Receive(timeout int)** — polls for incoming messages
   - JSON-RPC method: `receive`
   - Params: `{"timeout": <seconds>}`
   - Returns array of message envelopes
   - NOTE: This may fail if daemon is in auto-receive mode. If `receive` doesn't work, investigate alternative approaches (the implementing agent should check signal-cli docs and experiment with the running daemon)

### Message types to model

```
SignalGroup: ID, Name, Members[]
SignalMessage: SourceNumber, SourceName, Timestamp, Text, GroupID, GroupName
SendResult: Timestamp, Results[] (RecipientAddress, Status)
```

### Error handling

- Return structured errors wrapping JSON-RPC error responses
- Handle connection refused (daemon not running) gracefully with clear error message
- HTTP timeout of 30 seconds for send operations, configurable timeout for receive

### Tests

- Unit tests with mock HTTP server simulating signal-cli JSON-RPC responses
- Test successful send, receive, list groups
- Test error handling (connection refused, JSON-RPC errors, malformed responses)
