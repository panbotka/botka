Tool calls disappear after the AI response finishes streaming. They should be preserved and viewable by clicking/tapping on the assistant message.

## Current behavior
- During streaming, tool calls are shown inline (ToolCallPanel components)
- Once the response completes, tool calls vanish — they're only in streaming state, not persisted
- There's no way to see what tools Claude used after the fact

## Expected behavior
- Tool calls are saved and associated with the assistant message
- After streaming completes, tool calls are NOT visible by default (clean UI)
- Clicking/tapping on an assistant message reveals the tool calls used during that response
- Tool calls shown in a collapsible section below the message text

## Requirements

### Backend — Persist tool calls
- Save tool call data with each assistant message. Options:
  - A `tool_calls` JSON column on the `messages` table (simplest — stores array of tool call objects)
  - Or a separate `tool_calls` table with `message_id` foreign key
- Each tool call record: `name` (string), `input` (JSON string or object), `output` (string, if available from tool_result events)
- During streaming in `chat.go`, collect tool calls (already parsed from `tool_use` events) and save them when creating the assistant message
- Include tool calls in the message API response

### Frontend — Collect tool calls during streaming
- During streaming, the tool calls are already tracked in SSEContext or ChatView state
- When the response completes, pass the collected tool calls to the message save/update flow
- After streaming ends and the message is persisted, the tool calls should come from the API (persisted data), not from streaming state

### Frontend — Show tool calls on click
- Assistant messages get a subtle indicator that tool calls are available (e.g. a small icon or "N tool calls" text, muted)
- Clicking/tapping the message (or the indicator) toggles a collapsible section showing the tool calls
- Use the existing `ToolCallPanel` component (or similar) for displaying each tool call
- The section should be below the message text, with a smooth expand/collapse animation
- Default state: collapsed

### Frontend — Indicator
- Show a small muted line below assistant messages that used tools, e.g.:
  - "3 tool calls" or a small wrench icon + count
  - Only visible for messages that have tool calls
  - Clickable — toggles the tool call detail section

### Data format
- Tool calls JSON structure per message:
  ```json
  [
    {"name": "Read", "input": {"file_path": "/home/pi/foo.go"}},
    {"name": "Bash", "input": {"command": "make test"}},
    {"name": "Edit", "input": {"file_path": "...", "old_string": "...", "new_string": "..."}}
  ]
  ```
- Tool result/output is optional — include if easily available from the stream, skip if not

## Implementation Notes
- The streaming already parses tool_use events with name and input — just collect them in a slice
- In `chat.go` when saving the assistant message, add the tool_calls JSON
- Migration needed: add `tool_calls` JSONB column to `messages` table (nullable)
- Message API serialization: include `tool_calls` in the JSON response
- Frontend: the `Message` type needs a `tool_calls` field
- Keep the collapsed/expanded state local (per-message `useState`), don't persist it
- Existing tests must pass (`make check`)
