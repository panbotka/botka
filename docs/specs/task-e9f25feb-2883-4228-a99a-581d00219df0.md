## Problem

The Settings page (`/settings`) has broken dark mode styling. Active toggle buttons (theme, font size) use `bg-zinc-900 text-white` which in dark mode renders as white text on a nearly-black background that blends with the dark mode page background — effectively invisible.

Similarly, inactive buttons use `bg-zinc-100 text-zinc-600` which is light-mode only.

Labels use hardcoded `text-zinc-700` which is invisible in dark mode.

The select dropdown for Default AI Model also uses light-only colors (`border-zinc-300 bg-zinc-50 text-zinc-900`).

## File

`frontend/src/pages/SettingsPage.tsx` — the `GeneralTab` component (lines ~92-199)

## What to Fix

1. **Active buttons** (theme picker, font size): `bg-zinc-900 text-white` → add `dark:bg-white dark:text-zinc-900`
2. **Inactive buttons**: `bg-zinc-100 text-zinc-600 hover:bg-zinc-200` → add `dark:bg-zinc-700 dark:text-zinc-300 dark:hover:bg-zinc-600`
3. **Labels**: `text-zinc-700` → add `dark:text-zinc-300`
4. **Select dropdown**: add dark variants for border, background, and text
5. **Toggle switches**: `bg-zinc-300` (off state) → add `dark:bg-zinc-600`

Scan the entire SettingsPage.tsx (all tabs: General, Personas, Tags, Memories, Voice) for any other hardcoded light-mode-only colors and fix them too.
