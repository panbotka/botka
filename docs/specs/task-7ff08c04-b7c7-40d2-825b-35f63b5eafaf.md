Refocus the /cost dashboard page on token consumption per model instead of dollar cost. The user has a monthly subscription so cost isn't the primary metric — token usage and rate limit consumption per model (opus, sonnet, haiku) is what matters.

## Current state

**Backend** (`internal/handlers/analytics.go`):
- `GET /api/v1/analytics/cost` returns: `total_cost_usd`, `by_date` (with cost_usd, input_tokens, output_tokens), `by_thread` (cost_usd), `by_project` (cost_usd)
- No model breakdown anywhere
- Messages join on threads which have a `model` field — this is how to get per-model data

**Frontend** (`frontend/src/pages/CostDashboardPage.tsx`):
- Three summary cards: Total Spend, Daily Average, Today — all showing dollar amounts
- Daily spend bar chart in dollars
- Top threads and projects by cost
- Everything is cost-centric

**Data model:**
- `messages` table: has `prompt_tokens`, `completion_tokens`, `cost_usd` — no `model` column
- `threads` table: has `model` (nullable, e.g. "opus", "sonnet", "haiku")
- `task_executions` table: has `cost_usd` — no `model`, no token columns
- Tasks don't have model info, but tasks table has no model field either

## Requirements

### Backend — Add model breakdown to analytics endpoint
- Add `by_model` array to the analytics response:
  ```json
  "by_model": [
    {"model": "opus", "input_tokens": 500000, "output_tokens": 120000, "cost_usd": 1.23, "message_count": 45},
    {"model": "sonnet", "input_tokens": 200000, "output_tokens": 50000, "cost_usd": 0.30, "message_count": 120},
    {"model": "haiku", "input_tokens": 50000, "output_tokens": 10000, "cost_usd": 0.02, "message_count": 15}
  ]
  ```
- Query: JOIN messages with threads on thread_id, GROUP BY threads.model
- Include messages where thread model is NULL under "unknown"
- Also add model breakdown to `by_date`:
  ```json
  "by_date": [{"date": "2026-03-25", "cost_usd": 0.5, "input_tokens": 100000, "output_tokens": 20000, "by_model": {"opus": {"input": 80000, "output": 15000}, "sonnet": {"input": 20000, "output": 5000}}}]
  ```
- Add totals for tokens: `total_input_tokens` and `total_output_tokens` at the top level

### Frontend — Redesign summary cards
- Replace dollar-focused cards with token-focused cards:
  - **Total Tokens**: sum of all input + output tokens (big number, formatted like "1.2M" or "450K")
  - **By Model**: show each model with its token count (opus: 500K in / 120K out, sonnet: 200K in / 50K out, etc.)
  - Keep cost as secondary info (smaller text, not the headline)

### Frontend — Token chart by model
- The daily chart should show token usage (not dollars) as stacked bars or grouped bars per model
- Each model gets a distinct color (e.g. opus = purple/violet, sonnet = blue, haiku = green)
- Tooltip shows breakdown: "opus: 80K in, 15K out / sonnet: 20K in, 5K out"
- Keep it simple — stacked CSS bars work fine

### Frontend — Model breakdown section
- Add a "By Model" section showing:
  - Each model with total input/output tokens and percentage of total
  - Visual bar showing proportion
  - Message count per model

### Frontend — Keep existing sections
- Keep Top Threads and Top Projects but show tokens alongside cost (e.g. "450K tokens / $0.42")

### Token formatting
- Use human-readable format: "1.2M", "450K", "12K" for large numbers
- Show exact numbers in tooltips
