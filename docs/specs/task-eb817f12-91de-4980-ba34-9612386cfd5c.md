Dashboard stat cards have inconsistent heights — some have subtitles, some don't, and value text length varies. Cards in the same grid row should all be the same height (matching the tallest one).

## Current state

**`DashboardPage.tsx`** has multiple grid rows of `StatCard` components:
- Line ~428: `grid grid-cols-2 gap-4 lg:grid-cols-4` — Running, Queued, Done, Failed
- Line ~464: `grid grid-cols-2 gap-4 lg:grid-cols-4` — Needs Review, Pending, Total Tasks, Success Rate
- Line ~501: `grid grid-cols-1 gap-4 sm:grid-cols-3` — Avg Duration, Total Cost, Most Active Project

**`StatCard`** (line ~69): The outer div has no height constraint. Cards with `subtitle` are taller than those without. Long values (like "chatovadlo") make cards wider but can also affect height with wrapping.

## Fix

Add `h-full` to the `StatCard` outer div so CSS Grid's default `align-items: stretch` makes all cards in a row the same height. The grid already handles this — the cards just need to fill their grid cell.

### Changes
- In `StatCard` (line ~71): add `h-full` to the outer div's className
- That's it — CSS Grid stretches cells to row height by default, the cards just need `h-full` to fill the cell
- If wrapped in a `<Link>`, the Link also needs `h-full` (or `block` display) so the card fills the link container
